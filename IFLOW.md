## 二、分层职责速查

| 层级 | 职责               | 关键点                                       |
| ---- | ------------------ | -------------------------------------------- |
| APP  | 业务逻辑、任务调度 | 唯一使用 FreeRTOS 的层，管理队列/互斥量/任务 |
| DRV  | 模块驱动、协议解析 | 封装外部模块，提供解析函数，不使用 RTOS      |
| BSP  | 外设封装、中断分发 | 管理 MCU 外设实例，回调分发，不配置硬件      |

## 三、重要原则

1. **中断中不解析数据**：只拷贝和传递，在任务上下文中解析
2. **单向依赖**：上层调用下层，下层不能调用上层

## 四、配置文件

### 4.1 配置文件职责

| 文件            | 位置       | 职责说明                                         |
| --------------- | ---------- | ------------------------------------------------ |
| **app_cfg.h**   | `app/Inc/` | 开发板选择、应用参数配置（任务频率、底盘参数等） |
| **bsp_cfg.h**   | `bsp/Inc/` | 板载资源定义、硬件映射、实例数量配置             |
| **drv_types.h** | `drv/`     | 跨层通用类型定义（不涉及硬件配置）               |

### 4.2 配置传递机制

```
┌─────────────────────────────────────────────────────────────┐
│  app_cfg.h（唯一修改点）                                     │
│  - DEVELOPMENT_BOARD 宏选择开发板                            │
│  - 应用参数（任务频率、底盘参数等）                          │
└─────────────────────────────────────────────────────────────┘
                          ↓ 包含
┌─────────────────────────────────────────────────────────────┐
│  bsp_cfg.h（自动适配）                                       │
│  - 根据 DEVELOPMENT_BOARD 自动配置实例数量                   │
│  - 定义板载资源枚举（GPIO/TIM/USART）                        │
│  - 声明硬件映射数组                                          │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 切换开发板

**只需修改 `app/Inc/app_cfg.h` 中的一个宏：**

```c
// app/Inc/app_cfg.h

#define STM32F407VET6 0
#define DM_MC02 1
#define DJI_A 2
#define DJI_C 3

// ⬇️ 修改此处切换开发板 ⬇️
#define DEVELOPMENT_BOARD STM32F407VET6
```

**bsp_cfg.h 会自动适配：**

- 实例数量配置
- 硬件映射数组
- CPU频率等硬件参数
