# 配置文件

## 一、概述

项目配置文件有**两种角色**的宏，按用途分到不同文件：

| 角色        | 说明                                                     | 存放位置                                                      |
| ----------- | -------------------------------------------------------- | ------------------------------------------------------------- |
| **功能1**   | 被 C/H 源码引用（`#ifdef`、`#if`、数值替换）             | `app_cfg.h`、`bsp.h`、`bsp_map.h`、`robot_def.h`              |
| **功能2**   | 被 `CMakeLists.txt` 识别（`read_define`/`check_define`） | `user_cfg.h`、`app_cfg.h`                                     |
| **功能1+2** | 两者都用                                                 | `app_cfg.h`（`DEVELOPMENT_BOARD`）、`bsp_map.h`（`CPU_CORE`） |

---

## 二、配置文件清单

### 2.1 `user_cfg.h` — 用户配置入口（根目录）

**角色**：功能2（仅 CMake 读取）

拉取代码后**可能需要修改**的文件，选择编译哪个应用和构建类型。

```c
#define APP_NAME  target_car   // 选择应用，对应 app/<APP_NAME>/ 目录
#define BUILD_TYPE Debug       // Debug / Release
```

**CMake 读取方式**：

| 宏           | CMake 函数               | 用途                                                     |
| ------------ | ------------------------ | -------------------------------------------------------- |
| `APP_NAME`   | `read_define()`          | 选择 `app/<APP_NAME>/` 目录，加载对应的 `CMakeLists.txt` |
| `BUILD_TYPE` | `file(STRINGS)` 正则提取 | 设置 `CMAKE_BUILD_TYPE`                                  |

---

### 2.2 `app/<APP>/app_cfg.h` — 应用配置文件

**角色**：功能1 + 功能2 + 功能1+2

这是**唯一被 BSP/DRV/APP 层包含**的配置头文件。底层代码通过 `#include "app_cfg.h"` 获取所有配置。

**包含链**：`app_cfg.h` → `bsp_map.h` → `bsp.h`

```c
#include "bsp_map.h"     // 硬件映射（提供 DM_MC02 等板常量、ITCM_RAM 等属性宏）
#include "robot_def.h"   // 机器人参数（用户自定义）

// === 功能开关（功能1：C 源码 #ifdef 条件编译）===
#define BSP_DWT_USED       // DWT 高精度定时器
#define BSP_GPIO_USED      // GPIO 管理 + EXTI 分发
#define BSP_MATH_USED      // 数学库（向量/矩阵/四元数/三角函数）
#define BSP_FREERTOS_USED  // FreeRTOS 静态创建封装
#define BSP_SPI_USED       // SPI (BMI088 / LCD)
#define BSP_TIM_USED       // 定时器 (PWM / 编码器)
#define BSP_USART_USED     // 串口 (SBUS / VOFA / RS485)
#define BSP_ADC_USED       // ADC (电池电压)
#define BSP_CAN_USED       // CAN (FDCAN / BXCAN, DJI 电机)
#define DRV_PID_USED       // PID 控制器
#define DRV_MIT_USED       // MIT PD 控制器
#define DRV_BMI088_USED    // BMI088 IMU 驱动
#define DRV_SBUS_USED      // SBUS 遥控器驱动
#define DRV_MAHONY_USED    // Mahony 姿态解算
#define DRV_CHASSIS_USED   // 底盘运动学
#define DRV_DJIMOTOR_USED  // DJI 电机驱动
#define BMI088_HEAT_USED   // BMI088 加热
#define DRV_AXIS_MIT_LITE_USED  // 单轴 MIT 关节控制
#define DAEMON_USED        // Daemon 看门狗
#define VOFA_LITE_USED     // VOFA+ JustFloat 遥测

// === 开发板选择（功能1+2）===
#define DEVELOPMENT_BOARD DM_MC02  // 切换开发板只需改这一行

// === CubeMX 配置（功能2：CMake 读取）===
#define HAL_CONFIG_NAME DM_MC02

// === 构建后处理开关（功能2：CMake check_define）===
#define GENERATE_DISASSEMBLY  // 生成反汇编 .lst
#define GENERATE_READELF      // 生成 readelf 输出
```

**CMake 读取方式**：

| 宏                     | CMake 函数       | 用途                                                                                                       |
| ---------------------- | ---------------- | ---------------------------------------------------------------------------------------------------------- |
| `DEVELOPMENT_BOARD`    | `read_define()`  | 选择开发板配置表 → 确定 CPU/FPU/链接脚本；同时 `add_compile_definitions(DEVELOPMENT_BOARD=...)` 传给编译器 |
| `HAL_CONFIG_NAME`      | `read_define()`  | 选择 `cubemx/<NAME>/` 目录                                                                                 |
| `GENERATE_DISASSEMBLY` | `check_define()` | 控制编译后是否生成 `.lst` 反汇编文件                                                                       |
| `GENERATE_READELF`     | `check_define()` | 控制编译后是否生成 readelf 输出文件                                                                        |

**切换开发板**：只需修改 `DEVELOPMENT_BOARD` 宏。支持的开发板：

| 宏值          | 主控芯片      | CAN 类型 |
| ------------- | ------------- | -------- |
| `DM_MC02` (0) | STM32H723VGT6 | FDCAN    |
| `DJI_A` (1)   | STM32F427IIH6 | BXCAN    |
| `DJI_C` (2)   | STM32F407IGH6 | BXCAN    |

### 2.3 `app/<APP>/robot_def.h` — 机器人参数定义

**角色**：功能1（仅 C 源码引用）

用户自定义的机器人参数，与框架配置分离。`app_cfg.h` 通过 `#include "robot_def.h"` 引入。

```c
#define v_speed 3.2f  // 最大线速度 (m/s)
#define w_speed 8.0f  // 最大角速度 (rad/s)
```

可根据需要扩展：底盘类型、PID 参数、任务频率、栈大小等。

---

### 2.4 `bsp/bsp.h` — 板级常量与工具宏

**角色**：功能1（仅 C 源码引用）

定义跨开发板共享的基础常量和工具宏，不包含任何硬件配置。

```c
// 开发板类型常量
#define DM_MC02 0
#define DJI_A   1
#define DJI_C   2

// 内核类型常量
#define CORTEX_M4 0
#define CORTEX_M7 1

// CAN 控制器类型常量
#define BSP_CAN_IP_BXCAN  0
#define BSP_CAN_IP_FDCAN  1

// 工具宏
#define BSP_ARRAY_SIZE(arr)  ...
#define BSP_STATIC_ASSERT_MAP_SIZE(arr, max)  ...
#define BSP_RETURN_IF_TRUE(cond, ret)  ...
#define BSP_RETURN_IF_TRUE_LOG(cond, ret, log_expr)  ...
```

**被包含方式**：`bsp_map.h` → `bsp.h`（不直接包含）

---

### 2.5 `bsp/bsp_map/<BOARD>/bsp_map.h` — 板级硬件映射

**角色**：功能1 + 功能1+2

每个开发板有独立的 `bsp_map.h`，CMake 根据 `DEVELOPMENT_BOARD` 选择对应目录加入 include 路径（`bsp/bsp_map/${BOARD}`）。

```c
#include "bsp.h"

// 硬件特性（功能1）
#define CPU_CORE CORTEX_M7        // 功能1+2：CMake 读取用于 ARM_MATH_CMx 定义
#define BSP_CAN_IP BSP_CAN_IP_FDCAN
#define HAS_FPU   1
#define HAS_DSP   1
#define HAS_CORDIC 1
#define HAS_CRC   1

// 内存属性宏（功能1）
#define DMA_RAM  __attribute__((section(".ram_d1")))
#define ITCM_RAM __attribute__((section(".itcmram")))

// 实例数量（功能1）
#define CAN_INSTANCE_NUM  24
#define SPI_INSTANCE_NUM  2
#define GPIO_INSTANCE_NUM 11
// ... 等

// 映射结构体类型和外设枚举
typedef struct { GPIO_TypeDef *port; uint16_t pin; } GPIO_Map_t;
// ... 等
```

**CMake 读取方式**：

| 宏         | CMake 函数      | 用途                                                       |
| ---------- | --------------- | ---------------------------------------------------------- |
| `CPU_CORE` | `read_define()` | 推导 `ARM_MATH_DEFINE`（`ARM_MATH_CM7` 或 `ARM_MATH_CM4`） |

---

### 2.6 `CMakeLists.txt` — 构建系统

**读取配置的两个辅助函数**：

```cmake
# 读取有值的宏（如 DEVELOPMENT_BOARD、HAL_CONFIG_NAME）
read_define(变量名 "文件路径" 宏名)

# 检测开关型宏（如 GENERATE_DISASSEMBLY），返回 ON/OFF
check_define(变量名 "文件路径" 宏名)
```

**配置指纹**：`APP_NAME.DEVELOPMENT_BOARD.HAL_CONFIG_NAME.BUILD_TYPE.CUBEMX_TIMESTAMP`

任何配置变更会自动清除 build 目录，确保增量编译正确。

## 三、依赖关系

```
CMake 读取:                       C 编译:

user_cfg.h                        app_cfg.h
  ├─ APP_NAME ─────────────────→ 选择 app 目录      ├─ #include "bsp_map.h" ──→ bsp_map.h ──→ bsp.h
  └─ BUILD_TYPE ───────────────→ Debug/Release     ├─ #include "robot_def.h" → 机器人参数
                                                    ├─ DEVELOPMENT_BOARD      (功能1+2)
app_cfg.h (CMake 读取):                             ├─ BSP_xxx_USED           (功能1)
  ├─ DEVELOPMENT_BOARD ────────→ 板配置/编译定义    ├─ DRV_xxx_USED           (功能1)
  ├─ HAL_CONFIG_NAME ──────────→ CubeMX 目录        ├─ GENERATE_*            (功能2)
  ├─ GENERATE_DISASSEMBLY ─────→ .lst 生成          └─ HAL_CONFIG_NAME       (功能2)
  └─ GENERATE_READELF ─────────→ readelf 输出
                                                    BSP/DRV .c:
bsp_map.h (CMake 读取):                               #include "app_cfg.h"  → 获取所有配置
  └─ CPU_CORE ─────────────────→ ARM_MATH_CMx
```

**关键原则**：
- `app_cfg.h` 是 BSP/DRV/APP 三层**唯一**包含的配置头
- `DEVELOPMENT_BOARD` 同时被 CMake 读取（选择板配置）和 C 编译使用（`#if` 条件选择）
- `user_cfg.h` 仅 CMake 读取，C 代码不包含

## 四、使用方式

### 4.1 拉取代码后

1. 编辑 `user_cfg.h`：设置 `APP_NAME`（选应用）、`BUILD_TYPE`（选 Debug/Release）
2. 编辑 `app/<APP_NAME>/app_cfg.h`：设置 `DEVELOPMENT_BOARD`（选开发板）
3. `cmake --preset default -B build && cmake --build build -j24`

### 4.2 切换开发板

修改 `app/<APP_NAME>/app_cfg.h` 中的一行：

```c
#define DEVELOPMENT_BOARD DM_MC02  // 改为 DJI_A 或 DJI_C
```

CMake 自动切换链接脚本、启动文件、CPU/FPU 标志、bsp_map 路径。

### 4.3 新建应用

```bash
# 1. 创建自己的 app 目录和 CMakeLists.txt
mkdir -p app/my_app
cp app/target_car/CMakeLists.txt app/my_app/

# 2. 写自己的 app_cfg.h（#include "bsp_map.h" 继承框架配置）
# 3. 写自己的 robot_def.h（机器人参数）
# 4. 写自己的业务代码

# 5. 修改 user_cfg.h
#define APP_NAME my_app

# 6. 编译
cmake --preset default -B build && cmake --build build -j24
```

### 4.4 关闭/开启模块

在 `app_cfg.h` 中注释/取消注释对应 `_USED` 宏：

```c
// #define BSP_CRC_USED    // 关闭 CRC 模块
#define BSP_CAN_USED       // 开启 CAN 模块
```

关闭后对应 BSP/DRV 源文件编译为空，链接时如被其他代码调用会报错（这是设计意图，作为配置错误检查）。

## 五、宏角色速查

| 宏                           | 所在文件      | 角色    | 说明                               |
| ---------------------------- | ------------- | ------- | ---------------------------------- |
| `APP_NAME`                   | `user_cfg.h`  | 功能2   | CMake 读取，选 app 目录            |
| `BUILD_TYPE`                 | `user_cfg.h`  | 功能2   | CMake 读取，选构建类型             |
| `DEVELOPMENT_BOARD`          | `app_cfg.h`   | 功能1+2 | C `#if` 比较 + CMake 板配置        |
| `HAL_CONFIG_NAME`            | `app_cfg.h`   | 功能2   | CMake 读取，选 CubeMX 目录         |
| `GENERATE_DISASSEMBLY`       | `app_cfg.h`   | 功能2   | CMake `check_define`，生成 .lst    |
| `GENERATE_READELF`           | `app_cfg.h`   | 功能2   | CMake `check_define`，生成 readelf |
| `BSP_xxx_USED`               | `app_cfg.h`   | 功能1   | C `#ifdef`，BSP 模块开关           |
| `DRV_xxx_USED`               | `app_cfg.h`   | 功能1   | C `#ifdef`，DRV 模块开关           |
| `LOG_GLOBAL_LIMIT`           | `app_cfg.h`   | 功能1   | C 数值替换，日志限流               |
| `v_speed` / `w_speed`        | `robot_def.h` | 功能1   | C 数值替换，车辆参数               |
| `CPU_CORE`                   | `bsp_map.h`   | 功能1+2 | C 条件编译 + CMake 读取            |
| 板类型常量 (`DM_MC02` 等)    | `bsp.h`       | 功能1   | C `#if ==` 比较                    |
| 工具宏 (`BSP_ARRAY_SIZE` 等) | `bsp.h`       | 功能1   | C 代码复用                         |
| `HAS_FPU` / `HAS_DSP` 等     | `bsp_map.h`   | 功能1   | C `#if` 硬件特性判断               |
| `CAN_INSTANCE_NUM` 等        | `bsp_map.h`   | 功能1   | C 数组大小定义                     |
| `DMA_RAM` / `ITCM_RAM`       | `bsp_map.h`   | 功能1   | C 内存段属性                       |
