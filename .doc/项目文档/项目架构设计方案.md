# 项目架构设计方案

本文档描述 test_my_frame 项目的分层架构设计原则、数据流机制和封装规范，适用于跨层设计参考。

---

## 一、整体架构

### 1.1 分层结构

```
┌─────────────────────────────────────────────────────────────┐
│                     APP 层（应用层）                         │
│  - 业务逻辑                                                  │
│  - 任务调度                                                  │
│  - 消息处理                                                  │
│  - FreeRTOS 对象管理（队列、互斥量、信号量）                 │
└─────────────────────────────────────────────────────────────┘
                          ↓ 调用 DRV 接口
┌─────────────────────────────────────────────────────────────┐
│                     DRV 层（驱动层）                         │
│  - 外部模块驱动（电机、IMU、遥控器等）                       │
│  - 协议解析                                                  │
│  - 数据处理                                                  │
│  - 回调传递                                                  │
└─────────────────────────────────────────────────────────────┘
                          ↓ 调用 BSP 接口
┌─────────────────────────────────────────────────────────────┐
│                     BSP 层（板级支持层）                     │
│  - MCU 外设封装（CAN、UART、SPI、GPIO 等）                   │
│  - 中断回调分发                                              │
│  - 实例管理                                                  │
└─────────────────────────────────────────────────────────────┘
                          ↓ 调用 HAL 库
┌─────────────────────────────────────────────────────────────┐
│              HAL 层 + SEGGER RTT + FreeRTOS                  │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 层级调用原则

| 规则         | 说明                                     |
| ------------ | ---------------------------------------- |
| **单向依赖** | 上层调用下层，下层不能调用上层           |
| **禁止跨层** | APP 不能直接调用 HAL，必须通过 DRV → BSP |
| **接口隔离** | 每层只暴露必要的接口，隐藏实现细节       |
| **配置向下** | 上层配置驱动下层，下层不依赖上层配置文件 |

### 1.3 各层职责

| 层级 | 职责               | 关键点                                           |
| ---- | ------------------ | ------------------------------------------------ |
| APP  | 业务逻辑、任务调度 | 调用 DRV 接口，管理 RTOS 对象，使用DEF宏定义实例 |
| DRV  | 模块驱动、协议解析 | 封装外部模块，提供解析函数，值拷贝BSP实例        |
| BSP  | 外设封装、中断分发 | 定义DEF宏，管理实例数组，提供板级映射            |
| HAL  | 硬件抽象层         | 包含多开发板配置，CubeMX 生成的基础代码          |

## 二、信息流设计

### 2.1 中断数据流

中断发生时的数据流向（必须先注册，才能回调）：

```
[MPU] interrupt
   │
   ↓ HAL 回调函数
   │
[BSP] hal_callback
   │    └─ 拷贝原始数据，查找实例，调用其回调
   ↓
[DRV] bsp_callback
   │    └─ 继续回调到 APP 层
   ↓
[APP] drv_callback
   │    └─ 拷贝原始消息，塞入队列（不做解析！）
   ↓
(interrupt finish) 中断结束
```

**重要原则：中断中不解析数据，只拷贝和传递**

### 2.2 任务处理流

消息处理线程中的数据流向：

```
[APP] 从队列获取原始消息
   │
   ↓ 解析数据（协议解析在此进行）
   │
[APP] 更新模块状态，执行业务逻辑
   │
   ↓ 调用 DRV 层接口
   │
[DRV] 调用 BSP 层接口
   │
   ↓
[BSP] 调用 HAL 库操作硬件
```

### 2.3 完整数据流示例

以 CAN 接收电机数据为例：

```
┌─────────────────────────────────────────────────────────────┐
│                        中断上下文                            │
│                         (只拷贝，不解析)                     │
│                                                             │
│  CAN RX 中断触发                                            │
│       │                                                     │
│       ↓                                                     │
│  HAL_CAN_RxFifo0MsgPendingCallback()                        │
│       │                                                     │
│       ↓ 拷贝原始数据，查找实例                               │
│  bsp_can.c: CAN 实例回调                                     │
│       │                                                     │
│       ↓ 继续回调                                            │
│  drv_motor.c: DRV 层回调                                     │
│       │                                                     │
│       ↓ 继续回调                                            │
│  app_chassis.c: APP 层回调                                   │
│       │                                                     │
│       ↓ 拷贝原始帧到队列（不解析！）                         │
│  osMessageQueuePut(motor_queue, &raw_frame)                 │
│       │                                                     │
│       ↓                                                     │
│  中断结束                                                    │
└─────────────────────────────────────────────────────────────┘
                          │
                          ↓ 队列传递
┌─────────────────────────────────────────────────────────────┐
│                        任务上下文                            │
│                        (在此解析数据)                        │
│                                                             │
│  ChassisTask()                                              │
│       │                                                     │
│       ↓ 从队列获取原始帧                                    │
│  osMessageQueueGet(motor_queue, &raw_frame, NULL, timeout)  │
│       │                                                     │
│       ↓ 解析数据（协议解析）                                │
│  Motor_DecodeFrame(&raw_frame, &motor_data)                 │
│       │                                                     │
│       ↓ 更新电机状态                                        │
│  Motor_UpdateState(&motor_data)                             │
│       │                                                     │
│       ↓ 执行控制算法                                        │
│  PID 计算、状态机处理                                        │
│       │                                                     │
│       ↓ 调用 DRV 接口发送命令                               │
│  Motor_SendCommand()                                        │
└─────────────────────────────────────────────────────────────┘
```

## 三、面向对象封装原则

### 3.1 核心原则

| 原则               | 实现方式                     | 目的         |
| ------------------ | ---------------------------- | ------------ |
| **接口暴露最小化** | 只在 .h 中声明必要的公开函数 | 隐藏实现细节 |
| **禁止全局变量**   | 使用 static 限定文件作用域   | 减少耦合     |
| **线程安全通信**   | 使用队列、信号量、互斥量     | 避免竞态条件 |

### 3.2 文件组织规范

```
模块目录/
├── module.h          # 公开接口声明 + 结构体定义
├── module.c          # 实现 + 静态函数
└── module_def.h      # 类型定义（可选，供内部使用）
```

**module.h 示例**：

```c
#ifndef MODULE_H
#define MODULE_H

#include <stdint.h>

// 实例结构体定义（在.h中定义，方便调试）
typedef struct ModuleInstance
{
    float value;
    float ref;
    // ... 其他成员
} ModuleInstance;

// 公开接口
ModuleInstance *ModuleInit(void);
void ModuleSetRef(ModuleInstance *inst, float ref);

#endif
```

**module.c 示例**：

```c
#include "module.h"
#include "bsp_log.h"
#include <stdlib.h>
#include <string.h>

// ========== 私有变量（文件作用域）==========
static ModuleInstance *s_instances[MODULE_MAX_CNT] = {NULL};
static uint8_t s_instance_cnt = 0;

// ========== 公开接口实现 ==========
ModuleInstance *ModuleInit(void)
{
    if (s_instance_cnt >= MODULE_MAX_CNT) {
        LOGERROR("[module] Instance count exceeded");
        return NULL;
    }

    ModuleInstance *inst = malloc(sizeof(ModuleInstance));
    memset(inst, 0, sizeof(ModuleInstance));

    s_instances[s_instance_cnt++] = inst;
    return inst;
}
```

## 四、实例注册机制

### 4.1 设计思想

采用"DEF宏定义实例 + 注册函数"模式，实现静态分配：

```c
// APP层：使用DEF宏定义实例
DEF_MOTOR(motor1, CAN_ID_0x201);

// 注册到BSP层
MotorRegister(&motor1);

// 直接访问成员
motor1.ref = 100.0f;
float speed = motor1.speed;
```

### 4.2 注册机制实现模板

**头文件**（DEF宏定义在BSP层）：

```c
// motor.h
#ifndef MOTOR_H
#define MOTOR_H

// 实例结构体定义
typedef struct MotorInstance
{
    uint32_t can_id;
    uint8_t motor_type;

    // 测量值
    float angle;
    float speed;
    float current;

    // 控制器
    float ref;

    // 回调
    void (*callback)(struct MotorInstance *inst);
} MotorInstance;

// DEF宏：定义在BSP层
#define DEF_MOTOR(name, id)                    \
    static MotorInstance name = {              \
        .can_id = id,                          \
        .motor_type = 0,                       \
        .angle = 0,                            \
        .speed = 0,                            \
        .current = 0,                          \
        .ref = 0,                              \
        .callback = NULL,                      \
    };

// 注册函数
bool MotorRegister(MotorInstance *inst);
void MotorEnable(MotorInstance *inst);

#endif
```

**源文件**（静态数组，无malloc）：

```c
// motor.c
#include "motor.h"

// 静态实例数组（不使用指针数组）
static MotorInstance s_motors[MOTOR_MAX_CNT];
static uint8_t s_motor_cnt = 0;

// 注册函数：值拷贝
bool MotorRegister(MotorInstance *inst)
{
    if (s_motor_cnt >= MOTOR_MAX_CNT) {
        return false;  // 数量限制
    }

    s_motors[s_motor_cnt++] = *inst;  // 值拷贝
    return true;
}
```

## 五、配置传递机制

### 5.1 核心原则

**配置方向：上层配置驱动下层，下层不依赖上层配置文件**

```
┌─────────────────────────────────────────────────────────────┐
│                     APP 层                                   │
│  - app_cfg.h 定义开发板选择和应用参数                        │
│  - 通过初始化参数传递配置到下层                              │
└─────────────────────────────────────────────────────────────┘
                          ↓ 包含 app_cfg.h
┌─────────────────────────────────────────────────────────────┐
│                     BSP 层                                   │
│  - bsp_cfg.h 根据 DEVELOPMENT_BOARD 自动适配                 │
│  - 定义板载资源枚举、硬件映射、实例数量                       │
└─────────────────────────────────────────────────────────────┘
                          ↓ 无依赖
┌─────────────────────────────────────────────────────────────┐
│                     DRV 层                                   │
│  - drv_types.h 定义跨层通用类型                              │
│  - 不涉及硬件配置                                            │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 配置文件职责

| 配置文件      | 位置       | 职责说明                                     |
| ------------- | ---------- | -------------------------------------------- |
| `app_cfg.h`   | `app/Inc/` | 开发板选择、应用参数（任务频率、底盘参数等） |
| `bsp_cfg.h`   | `bsp/Inc/` | 板载资源枚举、硬件映射数组、实例数量配置     |
| `drv_types.h` | `drv/`     | 跨层通用类型定义（不涉及硬件配置）           |

### 5.3 切换开发板

**只需修改 `app/Inc/app_cfg.h` 中的一个宏：**

```c
// app/Inc/app_cfg.h

#define TELESKY_VET6 0
#define DM_MC02 1
#define DJI_A 2
#define DJI_C 3

// ⬇️ 修改此处切换开发板 ⬇️
#define DEVELOPMENT_BOARD TELESKY_VET6
```

**bsp_cfg.h 会自动适配：**

- 实例数量配置（GPIO_NUM, PWM_NUM 等）
- 硬件映射数组（gpio_map, tim_map, usart_map）
- CPU频率等硬件参数

### 5.4 配置传递示例

**错误方式**（下层依赖上层）：

```c
// ❌ drv层包含app_cfg.h
#include "app_cfg.h"  // 违反单向依赖原则
```

**正确方式**（参数传递）：

```c
// app_cfg.h - APP层定义配置
#define MOTOR_NUM 4

// app_motor.c - APP层传递配置
void MotorInit(void)
{
    for (int i = 0; i < MOTOR_NUM; i++) {
        DCMotor_Register(&motor_configs[i]);  // 通过参数传递
    }
}

// drv_dcmotor.c - DRV层从参数获取配置
DCMotorInstance *DCMotor_Register(DCMotor_Config_t *config)
{
    // 从config参数获取配置，而不是从app_cfg.h
    inst->param = config->param;
}
```

## 六、RTOS 接口规范

### 6.1 核心原则

**APP 层统一使用 FreeRTOS 原生 API，不使用 CMSIS-RTOS 封装层**

### 6.2 选择理由

| 理由 | 说明 |
|------|------|
| 静态分配更直接 | `xTaskCreateStatic()` 一步完成，无需宏展开 |
| 消息队列更灵活 | 原生支持任意大小结构体 |
| 无中间层开销 | CMSIS-RTOS 是封装层，底层仍调用 FreeRTOS |
| 官方支持更好 | FreeRTOS 文档丰富，社区活跃 |

### 6.3 API 对照表

| 功能 | FreeRTOS 原生 API |
|------|-------------------|
| 创建任务（静态） | `xTaskCreateStatic()` |
| 删除任务 | `vTaskDelete()` |
| 延时 | `vTaskDelay(pdMS_TO_TICKS(ms))` |
| 创建队列（静态） | `xQueueCreateStatic()` |
| 发送队列 | `xQueueSend()` |
| 接收队列 | `xQueueReceive()` |
| 创建互斥量（静态） | `xSemaphoreCreateMutexStatic()` |
| 获取互斥量 | `xSemaphoreTake()` |
| 释放互斥量 | `xSemaphoreGive()` |

### 6.4 类型定义对照

| CMSIS-RTOS v1 | FreeRTOS 原生 |
|---------------|---------------|
| `osThreadId` | `TaskHandle_t` |
| `osStaticThreadDef_t` | `StaticTask_t` |
| `osDelay(ms)` | `vTaskDelay(pdMS_TO_TICKS(ms))` |

## 七、DEF宏定义规范

### 6.1 核心原则

**DEF宏定义在底层（BSP层），APP层调用宏创建实例**

```
┌─────────────────────────────────────────────────────────────┐
│                     APP 层                                   │
│  - 调用DEF宏创建实例                                         │
│  - 调用注册函数将实例注册到BSP层                             │
│                                                              │
│  DEF_GPIO(motor_in1, GPIOB, GPIO_PIN_12);                   │
│  GPIORegister(&motor_in1);                                  │
└─────────────────────────────────────────────────────────────┘
                          ↑ 调用宏
┌─────────────────────────────────────────────────────────────┐
│                     BSP 层                                   │
│  - 定义DEF宏模板                                             │
│  - 提供注册函数接口                                          │
│  - 管理静态实例数组                                          │
│                                                              │
│  #define DEF_GPIO(name, port, pin) ...                      │
│  bool GPIORegister(GPIOInstance *instance);                 │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 DEF宏实现模板

**BSP层定义宏**（bsp_gpio.h）：

```c
// bsp_gpio.h - DEF宏定义在BSP层

#ifndef BSP_GPIO_H
#define BSP_GPIO_H

#include "main.h"

// 实例结构体定义
typedef struct {
    GPIO_TypeDef *GPIOx;
    uint16_t GPIO_Pin;
    GPIO_PinState state;
} GPIOInstance;

// DEF宏：定义在BSP层，供上层调用
#define DEF_GPIO(name, port, pin)              \
    static GPIOInstance name = {               \
        .GPIOx = port,                         \
        .GPIO_Pin = pin,                       \
        .state = GPIO_PIN_RESET,               \
    };

// 注册函数声明
bool GPIORegister(GPIOInstance *instance);

#endif
```

**APP层使用宏**（app_motor.c）：

```c
// app_motor.c - APP层调用宏创建实例

#include "bsp_gpio.h"

// 使用DEF宏定义实例（静态分配）
DEF_GPIO(motor_in1, GPIOB, GPIO_PIN_12);
DEF_GPIO(motor_in2, GPIOB, GPIO_PIN_13);

void MotorInit(void)
{
    // 注册到BSP层
    if (!GPIORegister(&motor_in1)) {
        // 注册失败处理
    }
    if (!GPIORegister(&motor_in2)) {
        // 注册失败处理
    }
}
```

### 6.3 DEF宏优势

| 特性       | 说明                                      |
| ---------- | ----------------------------------------- |
| 静态分配   | 编译时确定内存，无运行时开销              |
| 类型安全   | 宏展开后是结构体初始化，编译器可检查类型  |
| 数量限制   | BSP层数组大小限制实例数量，注册时检查     |
| 无动态内存 | 不使用 malloc，符合 FreeRTOS 静态分配要求 |

## 七、DRV访问BSP方式

### 7.1 核心原则

**DRV结构体直接包含BSP结构体（值拷贝），不使用指针**

### 7.2 对比说明

| 方式          | 定义                  | 生命周期    | 内存安全 |
| ------------- | --------------------- | ----------- | -------- |
| 指针方式 ❌   | `GPIOInstance *gpio;` | 需手动管理  | 悬空风险 |
| 值拷贝方式 ✅ | `GPIOInstance gpio;`  | 跟随DRV实例 | 安全     |

### 7.3 实现示例

**错误方式**（使用指针）：

```c
// ❌ drv_dcmotor.h - 使用指针
typedef struct DCMotorInstance
{
    GPIOInstance *gpio_in1;   // 指针，需外部管理生命周期
    GPIOInstance *gpio_in2;
    PWMInstance *pwm;
    EncoderInstance *encoder;
} DCMotorInstance;
```

**正确方式**（值拷贝）：

```c
// ✅ drv_dcmotor.h - 值拷贝
typedef struct DCMotorInstance
{
    GPIOInstance gpio_in1;    // 直接包含结构体
    GPIOInstance gpio_in2;
    PWMInstance pwm;
    EncoderInstance encoder;
} DCMotorInstance;

// 初始化时拷贝
void DCMotor_Init(DCMotorInstance *inst, DCMotor_Config_t *config)
{
    inst->gpio_in1 = config->gpio_in1;  // 结构体赋值，值拷贝
    inst->gpio_in2 = config->gpio_in2;
    inst->pwm = config->pwm;
    inst->encoder = config->encoder;
}
```

### 7.4 值拷贝优势

| 优势       | 说明                                       |
| ---------- | ------------------------------------------ |
| 内存安全   | DRV实例销毁时，BSP实例同时销毁，无悬空指针 |
| 无动态分配 | 不需要 malloc/free                         |
| 线程安全   | 每个DRV实例拥有独立的BSP实例副本           |
| 简单可靠   | 不需要管理指针生命周期                     |

### 7.5 注意事项

**BSP实例不能共用**：一个BSP实例只能被一个DRV实例使用（值拷贝后各自独立）

```c
// ❌ 错误：多个DRV实例共用同一个BSP实例
GPIOInstance shared_gpio;
DEF_GPIO(gpio1, GPIOB, GPIO_PIN_12);

// 注册到多个DRV实例会导致数据不一致
motor1.gpio_in1 = gpio1;  // 值拷贝
motor2.gpio_in1 = gpio1;  // 值拷贝，但两者独立，状态不同步

// ✅ 正确：每个DRV实例使用独立的BSP实例
DEF_GPIO(motor1_in1, GPIOB, GPIO_PIN_12);
DEF_GPIO(motor2_in1, GPIOB, GPIO_PIN_13);
```

## 八、回调注册与分发

### 5.1 回调注册流程

```
APP 调用 DRV 注册函数
       │
       ↓
DRV 调用 BSP 注册函数，传入回调
       │
       ↓
BSP 保存实例和回调指针
       │
       ↓
中断发生时，BSP 查找实例并调用回调
```

### 5.2 三层回调示例

**BSP 层回调分发**：

```c
// bsp_can.c

static CANInstance *s_can_instances[CAN_MAX_CNT] = {NULL};
static uint8_t s_can_cnt = 0;

CANInstance *CANRegister(CAN_Init_Config_s *config)
{
    CANInstance *inst = malloc(sizeof(CANInstance));
    inst->handle = config->handle;
    inst->rx_id = config->rx_id;
    inst->callback = config->callback;  // 保存上层回调
    inst->id = config->id;

    s_can_instances[s_can_cnt++] = inst;
    return inst;
}

void HAL_CAN_RxFifo0MsgPendingCallback(CAN_HandleTypeDef *hcan)
{
    CAN_RxHeaderTypeDef rx_header;
    uint8_t data[8];

    HAL_CAN_GetRxMessage(hcan, CAN_RX_FIFO0, &rx_header, data);

    for (uint8_t i = 0; i < s_can_cnt; i++) {
        CANInstance *inst = s_can_instances[i];

        if (inst->handle == hcan && rx_header.StdId == inst->rx_id) {
            inst->rx_len = rx_header.DLC;
            memcpy(inst->rx_buff, data, 8);

            if (inst->callback) {
                inst->callback(inst);  // 调用 DRV 层回调
            }
            break;
        }
    }
}
```

**DRV 层回调**：

```c
// drv_motor.c

static void MotorCANRxCallback(CANInstance *can_inst)
{
    MotorInstance *motor = (MotorInstance *)can_inst->id;

    // 继续回调到 APP 层，传递原始数据
    if (motor->app_callback) {
        motor->app_callback(can_inst);
    }
}
```

**APP 层回调**：

```c
// app_chassis.c

static void MotorDataCallback(CANInstance *can_inst)
{
    // 构造原始帧消息
    CanRawFrame_t raw_frame = {
        .can_id = can_inst->rx_id,
        .dlc = can_inst->rx_len,
        .timestamp = DWT_GetTimeline_us(),
    };
    memcpy(raw_frame.data, can_inst->rx_buff, 8);

    // 拷贝到队列（不解析！）
    osMessageQueuePut(s_chassis_queue, &raw_frame, 0, 0);
}
```

## 九、线程安全通信

### 6.1 队列创建规则

**重要原则：不同类型的队列在不同位置创建**

| 队列类型         | 创建位置          | 说明                     |
| ---------------- | ----------------- | ------------------------ |
| 线程间通信队列   | `app/Src/robot.c` | 用于任务间传递消息       |
| 中断到自己的队列 | 任务自己文件中    | 用于中断向本任务传递数据 |

**队列长度规则：**

- 队列长度统一设置为 **1**
- 原因：底盘等高频任务需要最新数据，旧数据无意义
- 使用 `osMessageQueueGet()` 阻塞等待即可

### 6.2 共享数据访问

**直接访问成员**：

```c
// motor.h - 结构体在.h中定义
typedef struct MotorInstance
{
    float speed;
    float angle;
    // ... 其他成员
} MotorInstance;

// app.c - 直接访问
float speed = motor->speed;
motor->ref = 100.0f;
```

**线程安全保护（需要时）**：

```c
// 在需要线程安全的场景下，使用互斥量保护
void MotorSetRef(MotorInstance *inst, float ref)
{
    osMutexAcquire(inst->mutex, osWaitForever);
    inst->ref = ref;
    osMutexRelease(inst->mutex);
}
```

### 6.3 使用队列传递数据

```c
// 定义消息类型（在 .h 中）
typedef struct {
    uint8_t motor_id;
    float angle;
    float speed;
} MotorMsg_t;

// 私有队列（在 .c 中）
static osMessageQueueId_t s_motor_queue = NULL;

// 初始化
void MotorInit(void)
{
    s_motor_queue = osMessageQueueNew(16, sizeof(MotorMsg_t), NULL);
}

// 发送（中断中）
osMessageQueuePut(s_motor_queue, &msg, 0, 0);

// 接收（任务中）
MotorMsg_t msg;
osStatus_t status = osMessageQueueGet(s_motor_queue, &msg, NULL, pdMS_TO_TICKS(10));
```

### 6.4 使用互斥量保护资源

```c
// 私有结构体
struct MotorInstance {
    // ...
    osMutexId_t mutex;  // 私有互斥量
};

// 创建实例时初始化互斥量
MotorInstance *MotorRegister(Motor_Init_Config_s *config)
{
    inst->mutex = osMutexNew(NULL);
}

// 访问接口中加锁
void MotorSetRef(MotorInstance *inst, float ref)
{
    osMutexAcquire(inst->mutex, osWaitForever);
    inst->ref = ref;
    osMutexRelease(inst->mutex);
}
```

## 十、通信方式选择

### 7.1 同步原语对比

| 通信方式       | 核心功能       | 关键特点       | 适用场景             |
| -------------- | -------------- | -------------- | -------------------- |
| **队列**       | 任务间传递消息 | 数据拷贝，FIFO | 中断到任务，大量数据 |
| **互斥量**     | 保护共享资源   | 优先级继承     | 共享数据访问         |
| **二值信号量** | 任务同步       | 轻量通知       | 事件通知             |
| **计数信号量** | 资源计数       | 多资源管理     | 资源池               |
| **任务通知**   | 直接通信       | 最轻量         | 任务间简单同步       |

### 7.2 典型应用场景

| 场景                    | 推荐方式   | 示例         |
| ----------------------- | ---------- | ------------ |
| 中断 → 任务（数据传递） | 队列       | CAN 接收数据 |
| 任务间共享数据          | 互斥量     | 电机状态读写 |
| 事件通知                | 二值信号量 | 数据就绪通知 |
| 高频数据最新值          | 覆盖式队列 | IMU 姿态数据 |

## 十二、总结

### 设计原则

1. **接口最小化**：只暴露必要的函数，隐藏实现细节
2. **作用域限定**：私有变量和函数使用 `static` 限定
3. **线程安全**：使用队列、互斥量等同步机制

### 数据流总结

```
中断：MPU → HAL → BSP → DRV → APP（拷贝原始帧入队列，不解析）
任务：APP（出队列）→ 解析数据 → 处理 → DRV → BSP → HAL
```

### 中断处理原则

| 层级 | 中断中操作             | 任务中操作             |
| ---- | ---------------------- | ---------------------- |
| BSP  | 拷贝原始数据、回调分发 | -                      |
| DRV  | 继续回调到 APP         | -                      |
| APP  | 拷贝原始帧到队列       | 解析数据、执行业务逻辑 |
