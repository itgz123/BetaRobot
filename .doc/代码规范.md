# 代码规范

本文档总结了 test_my_frame 项目的统一代码规范，适用于 BSP、DRV、APP 三层，供开发人员参考遵循。

---

## 一、命名规范

### 1.1 快速参考表

| 类别             | 规范                                 | 示例                                       |
| ---------------- | ------------------------------------ | ------------------------------------------ |
| **变量**         | snake_case（下划线分隔，全小写）     | `chassis_vx`, `motor_lf`                   |
| **函数**         | PascalCase（大驼峰）+ 模块前缀       | `DJIMotorInit()`, `ChassisTask()`          |
| **复杂结构体**   | `_s` 后缀                            | `Motor_Controller_s`, `Chassis_Ctrl_Cmd_s` |
| **简单数据类型** | `_t` 后缀                            | `attitude_t`, `DWT_Time_t`                 |
| **枚举类型**     | `_e` 后缀                            | `chassis_mode_e`, `Motor_Type_e`           |
| **实例类型**     | `Instance` 后缀                      | `DJIMotorInstance`, `CANInstance`          |
| **宏定义**       | UPPER_SNAKE_CASE（全大写下划线分隔） | `DJI_MOTOR_CNT`, `PI`                      |
| **文件**         | lower_case.h/c（全小写下划线分隔）   | `dji_motor.c`, `robot_def.h`               |

### 1.2 变量命名规范

#### 全局变量

- 使用 `static` 关键字修饰，限定在文件作用域内
- 命名风格：**snake_case**（全小写 + 下划线分隔）
- 通常不带特殊前缀

```c
// 示例
static Publisher_t *chassis_pub;
static Subscriber_t *chassis_sub;
static Chassis_Ctrl_Cmd_s chassis_cmd_recv;
static DJIMotorInstance *motor_lf, *motor_rf, *motor_lb, *motor_rb;
```

#### 局部变量

- 使用 **snake_case** 风格
- 简洁明了，采用有意义的缩写或全称
- 循环变量常用 `i`, `num`, `group` 等

```c
void DJIMotorControl()
{
    uint8_t group, num;
    int16_t set;
    float pid_measure, pid_ref;
}
```

#### 结构体成员变量

- 成员变量使用 **snake_case** 风格
- 指针成员使用 `_ptr` 后缀表示指针类型

```c
typedef struct
{
    float *other_angle_feedback_ptr;  // 角度反馈数据指针
    float *other_speed_feedback_ptr;  // 速度反馈数据指针
    float pid_ref;
} Motor_Controller_s;
```

#### 指针变量

- 指针变量命名通常**不带 `p` 前缀**
- 成员变量中的指针使用 `_ptr` 后缀

```c
static DJIMotorInstance *yaw_motor;
static attitude_t *gimba_IMU_data;
```

### 1.3 函数命名规范

#### 普通函数

- 使用 **大驼峰命名法（PascalCase）**
- 动宾短语结构
- 带模块前缀标识所属模块

```c
// 初始化函数
void RobotInit(void);
void ChassisInit(void);
DJIMotorInstance *DJIMotorInit(Motor_Init_Config_s *config);

// 任务函数
void RobotTask(void);
void ChassisTask(void);

// 操作函数
void DJIMotorSetRef(DJIMotorInstance *motor, float ref);
void DJIMotorEnable(DJIMotorInstance *motor);
```

#### 回调函数

- 回调函数以 `Callback` 为后缀
- HAL回调函数保持HAL库命名风格（下划线分隔）

```c
// 项目自定义回调函数
static void DJIMotorLostCallback(void *motor_ptr);

// HAL库回调函数（保持原有风格）
void HAL_CAN_RxFifo0MsgPendingCallback(CAN_HandleTypeDef *hcan);
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim);
```

#### 初始化函数

- 以 `Init` 为后缀
- 返回实例指针或 `void`
- 带模块前缀

```c
DJIMotorInstance *DJIMotorInit(Motor_Init_Config_s *config);
RC_ctrl_t *RemoteControlInit(UART_HandleTypeDef *rc_usart_handle);
```

#### 静态函数

- 使用 `static` 修饰
- 命名风格与普通函数相同（大驼峰）

```c
static void MotorSenderGrouping(DJIMotorInstance *motor, CAN_Init_Config_s *config);
static void DecodeDJIMotor(CANInstance *_instance);
```

### 1.4 类型定义命名规范

#### 结构体命名

| 后缀       | 用途                               | 示例                                             |
| ---------- | ---------------------------------- | ------------------------------------------------ |
| `_s`       | 复杂结构体（包含多个逻辑相关成员） | `Motor_Controller_s`, `Chassis_Ctrl_Cmd_s`       |
| `_t`       | 简单数据类型（数据载体）           | `attitude_t`, `DWT_Time_t`                       |
| `Instance` | 实例类型（运行时对象）             | `DJIMotorInstance`, `CANInstance`, `PIDInstance` |

```c
// 复杂结构体（_s 后缀）
typedef struct
{
    float vx;
    float vy;
    float wz;
    float offset_angle;
    chassis_mode_e chassis_mode;
} Chassis_Ctrl_Cmd_s;

// 简单数据类型（_t 后缀）
typedef struct
{
    float Gyro[3];
    float Accel[3];
    float Roll, Pitch, Yaw;
} attitude_t;

// 实例类型（Instance 后缀）- 结构体在.h文件中定义
typedef struct MotorInstance
{
    uint32_t rx_id;
    float speed;
    float angle;
} MotorInstance;
```

#### 枚举命名

- 使用 `_e` 后缀
- 枚举值使用**大写字母 + 下划线**
- 枚举值带统一前缀

```c
typedef enum
{
    CHASSIS_ZERO_FORCE = 0,
    CHASSIS_ROTATE,
    CHASSIS_NO_FOLLOW,
    CHASSIS_FOLLOW_GIMBAL_YAW,
} chassis_mode_e;

typedef enum
{
    OPEN_LOOP = 0b0000,
    CURRENT_LOOP = 0b0001,
    SPEED_LOOP = 0b0010,
    ANGLE_LOOP = 0b0100,
} Closeloop_Type_e;
```

#### 联合体命名

- 使用 `_t` 后缀
- 常用于位域操作

```c
typedef union
{
    struct
    {
        uint16_t w : 1;
        uint16_t s : 1;
        uint16_t d : 1;
        uint16_t a : 1;
        uint16_t shift : 1;
        uint16_t ctrl : 1;
    };
    uint16_t keys;
} Key_t;
```

### 1.5 宏定义命名规范

#### 普通宏定义

- **全大写字母 + 下划线分隔**（UPPER_SNAKE_CASE）
- 数值宏常带有类型后缀（如 `f` 表示 float）
- 带模块或功能前缀

```c
#define PI 3.1415926535f
#define PI2 (PI * 2.0f)
#define RAD_2_DEGREE 57.2957795f
#define DEGREE_2_RAD 0.01745329252f
#define DJI_MOTOR_CNT 12
#define SPEED_SMOOTH_COEF 0.85f
```

#### 条件编译宏

- 使用 `#ifndef` / `#define` / `#endif` 头文件保护
- 宏名为文件名的大写形式
- 或使用 `#pragma once`

```c
// 方式一：传统头文件保护
#ifndef MOTOR_DEF_H
#define MOTOR_DEF_H
// ...
#endif // !MOTOR_DEF_H

// 方式二：混合使用
#pragma once
#ifndef ROBOT_DEF_H
#define ROBOT_DEF_H
// ...
#endif // !ROBOT_DEF_H
```

#### 调试宏

- 日志宏以 `LOG` 为前缀
- 支持条件禁用

```c
#define LOGINFO(format, ...)  LOG_PROTO("I:", RTT_CTRL_TEXT_BRIGHT_GREEN, format, ##__VA_ARGS__)
#define LOGWARNING(format, ...) LOG_PROTO("W:", RTT_CTRL_TEXT_BRIGHT_YELLOW, format, ##__VA_ARGS__)
#define LOGERROR(format, ...) LOG_PROTO("E:", RTT_CTRL_TEXT_BRIGHT_RED, format, ##__VA_ARGS__)
```

#### 函数式宏

- 使用大写命名
- 参数用小写
- 使用括号保护参数
- 多语句宏使用 `do { } while(0)` 包装

```c
// 单语句宏
#define LIMIT_MIN_MAX(x, min, max) (x) = (((x) <= (min)) ? (min) : (((x) >= (max)) ? (max) : (x)))

// 多语句宏
#define TIME_ELAPSE(dt, code)                    \
    do                                           \
    {                                            \
        float tstart = DWT_GetTimeline_s();      \
        code;                                    \
        dt = DWT_GetTimeline_s() - tstart;       \
    } while (0)

// 简化判断宏
#define switch_is_down(s) (s == RC_SW_DOWN)
#define switch_is_mid(s) (s == RC_SW_MID)
#define switch_is_up(s) (s == RC_SW_UP)
```

### 1.6 文件命名规范

#### 分层文件命名规范

各层文件统一使用**层级前缀**命名，便于快速识别所属层级：

| 层级 | 文件前缀 | 示例 |
| ---- | -------- | ---- |
| APP  | `app_`   | `app_robot.h`, `app_chassis.c`, `app_cmd.h` |
| BSP  | `bsp_`   | `bsp_usart.h`, `bsp_gpio.c`, `bsp_tim.h` |
| DRV  | `drv_`   | `drv_motor.h`, `drv_pid.c`, `drv_imu.h` |

**命名规则：**
- 头文件：`层级前缀_模块名.h`
- 源文件：`层级前缀_模块名.c`
- 配置文件：`层级前缀_cfg.h`（如 `app_cfg.h`, `bsp_cfg.h`）
- 类型定义文件：`层级前缀_types.h`（如 `drv_types.h`）

#### 头文件

- **全小写 + 下划线分隔**
- 以 `.h` 为扩展名
- 必须带层级前缀

```
app_robot.h          # APP层 - 机器人总控
app_chassis.h        # APP层 - 底盘控制
app_cmd.h            # APP层 - 命令处理
bsp_usart.h          # BSP层 - 串口封装
drv_motor.h          # DRV层 - 电机驱动
```

#### 源文件

- **全小写 + 下划线分隔**
- 以 `.c` 为扩展名
- 与对应头文件同名

```
app_robot.c          # APP层 - 机器人实现
app_chassis.c        # APP层 - 底盘实现
app_cmd.c            # APP层 - 命令处理实现
bsp_usart.c          # BSP层 - 串口实现
drv_motor.c          # DRV层 - 电机驱动实现
```

### 1.7 命名前缀规范

#### 模块前缀

| 模块 | 前缀                 | 示例                                |
| ---- | -------------------- | ----------------------------------- |
| 电机 | `DJIMotor` / `Motor` | `DJIMotorInit()`, `MotorRegister()` |
| CAN  | `CAN`                | `CANInit()`, `CANRegister()`        |
| 底盘 | `Chassis`            | `ChassisInit()`, `ChassisTask()`    |
| 云台 | `Gimbal`             | `GimbalInit()`, `GimbalTask()`      |

#### 功能后缀

| 后缀                 | 用途         | 示例                                      |
| -------------------- | ------------ | ----------------------------------------- |
| `Init`               | 初始化函数   | `RobotInit()`, `ChassisInit()`            |
| `Task`               | 周期任务函数 | `RobotTask()`, `ChassisTask()`            |
| `Callback`           | 回调函数     | `DJIMotorLostCallback()`                  |
| `Register`           | 注册函数     | `MotorRegister()`, `DaemonRegister()`     |
| `Enable` / `Disable` | 使能/禁用    | `DJIMotorEnable()`, `DJIMotorStop()`      |
| `Get` / `Set`        | 获取/设置    | `DWT_GetTimeline_s()`, `DJIMotorSetRef()` |

### 1.8 单位规范

**所有变量、宏定义使用国际单位制（SI）**。

常用国际单位：

| 物理量   | 单位       | 示例                              |
| -------- | ---------- | --------------------------------- |
| 长度     | 米（m）    | `chassis_width = 0.4f`            |
| 速度     | 米/秒（m/s）| `max_speed = 3.0f`               |
| 角度     | 弧度（rad）| `yaw_angle = 1.57f`               |
| 角速度   | 弧度/秒（rad/s）| `angular_speed = 3.14f`      |
| 时间     | 秒（s）    | `dt = 0.001f`                     |
| 质量     | 千克（kg） | `robot_mass = 20.0f`              |
| 电流     | 安培（A）  | `motor_current = 5.0f`            |
| 温度     | 开尔文（K）或摄氏度（℃）| `motor_temp = 45.0f`    |

**注意**：如需使用其他单位（如度、rpm），应在命名中明确标注或在注释中说明。

```c
// 正确示例：使用国际单位
#define WHEEL_RADIUS 0.05f          // 单位：m
#define MAX_LINEAR_SPEED 3.0f       // 单位：m/s
float angle_rad = 1.57f;            // 单位：rad

// 特殊情况：需要使用其他单位时明确标注
#define RPM_TO_RAD_PER_SEC 0.10472f
float motor_speed_rpm = 300.0f;     // 单位：rpm（特殊需求，已标注）
float motor_speed_rad = motor_speed_rpm * RPM_TO_RAD_PER_SEC;  // 转换为国际单位
```

## 二、代码格式规范

### 2.1 缩进与空格

- 使用 **4 空格缩进**（不使用 Tab）
- 运算符两侧加空格
- 逗号后加空格
- 大括号独占一行

```c
// 正确示例
void ExampleFunction(int a, int b)
{
    int result = a + b;
    if (result > 0)
    {
        // do something
    }
}
```

### 2.2 大括号风格

- 使用 **Allman 风格**（大括号独占一行）

```c
// 正确
if (condition)
{
    // code
}
else
{
    // code
}

// 错误
if (condition) {
    // code
}
```

### 2.3 行长度限制

- 每行代码不超过 **100 字符**
- 过长的行应适当换行

### 2.4 空行规则

- 函数之间空一行
- 逻辑块之间空一行
- 文件末尾保留一个空行

## 三、注释规范

### 3.1 文件头注释

```c
/**
 * @file file_name.c
 * @brief 文件简要描述
 * @author 作者
 * @date 创建日期
 */
```

### 3.2 函数注释

```c
/**
 * @brief 简要描述函数功能
 *
 * @param param1 参数1说明
 * @param param2 参数2说明
 * @return 返回值说明
 *
 * @note 注意事项
 */
```

### 3.3 代码注释

- 注释应解释 **为什么** 而非 **是什么**
- 避免无意义的注释
- 使用英文注释（与代码风格一致）

```c
// 好的注释：解释原因
// 使用 DMA 传输避免阻塞主循环
HAL_UART_Transmit_DMA(&huart1, data, len);

// 不好的注释：重复代码含义
// 设置 speed 为 100
speed = 100;
```

## 四、编码规范

### 4.1 文件编码

- 统一使用 **UTF-8** 编码
- 换行符使用 **CRLF**（Windows 环境）

### 4.2 代码格式化

- 编写完成后使用 `Shift+Alt+F` 格式化代码

### 4.3 HAL 库代码规范

- 在 CubeMX 生成文件的 `USER CODE BEGIN` 和 `USER CODE END` 注释块内编写代码
- 不要修改外部代码

```c
void MX_GPIO_Init(void)
{
    /* USER CODE BEGIN MX_GPIO_Init_1 */
    // 你的代码
    /* USER CODE END MX_GPIO_Init_1 */

    // CubeMX 生成的代码...

    /* USER CODE BEGIN MX_GPIO_Init_2 */
    // 你的代码
    /* USER CODE END MX_GPIO_Init_2 */
}
```

### 4.4 编译警告处理

- 项目启用 `-Werror`，必须修复所有编译警告
- 不允许忽略警告

## 六、错误处理规范

### 6.1 参数检查

在关键位置进行参数验证：

```c
MotorInstance *MotorRegister(Motor_Init_Config_s *config)
{
    // 参数检查
    if (config == NULL)
    {
        LOGERROR("[motor] Config is NULL!");
        return NULL;
    }

    // 数量检查
    if (s_motor_cnt >= MOTOR_MAX_CNT)
    {
        LOGERROR("[motor] Instance count exceeded!");
        return NULL;
    }

    // ...
}
```

### 6.2 错误处理方式

| 错误类型   | 处理方式                        |
| ---------- | ------------------------------- |
| 初始化失败 | 返回 NULL，输出错误日志         |
| 参数无效   | 返回默认值或 NULL，输出警告日志 |
| 运行时错误 | 尝试恢复或进入安全状态          |

## 七、示例代码

### 完整示例：模块定义

```c
// motor.h - 电机模块定义

#ifndef MOTOR_H
#define MOTOR_H

#include <stdint.h>

// 宏定义（全大写）
#define MOTOR_MAX_CURRENT 16000.0f
#define MOTOR_MAX_SPEED 10000.0f

// 枚举（_e 后缀，值全大写）
typedef enum
{
    MOTOR_TYPE_M3508 = 0,
    MOTOR_TYPE_M6020,
    MOTOR_TYPE_M2006,
} Motor_Type_e;

// 实例结构体（Instance 后缀，在.h中定义）
typedef struct MotorInstance
{
    uint32_t rx_id;
    Motor_Type_e motor_type;
    float speed;
    float angle;
    float ref;
    void (*app_callback)(void *raw_data);
    void *app_id;
} MotorInstance;

// 配置结构体（_s 后缀）
typedef struct
{
    Motor_Type_e motor_type;
    uint32_t rx_id;
    void (*app_callback)(void *raw_data);
    void *app_id;
} Motor_Init_Config_s;

// 函数声明（大驼峰 + 模块前缀）
MotorInstance *MotorRegister(Motor_Init_Config_s *config);
void MotorEnable(MotorInstance *motor);
void MotorSetRef(MotorInstance *motor, float ref);

#endif // !MOTOR_H
```

```c
// motor.c - 电机模块实现

#include "motor.h"
#include "bsp_log.h"
#include <stdlib.h>
#include <string.h>

// 私有变量
static MotorInstance *s_motors[MOTOR_MAX_CNT] = {NULL};
static uint8_t s_motor_cnt = 0;

// 公开接口实现
MotorInstance *MotorRegister(Motor_Init_Config_s *config)
{
    if (config == NULL)
    {
        LOGERROR("[motor] Config is NULL!");
        return NULL;
    }

    if (s_motor_cnt >= MOTOR_MAX_CNT)
    {
        LOGERROR("[motor] Instance count exceeded!");
        return NULL;
    }

    MotorInstance *inst = malloc(sizeof(MotorInstance));
    if (inst == NULL)
    {
        LOGERROR("[motor] Memory allocation failed!");
        return NULL;
    }
    memset(inst, 0, sizeof(MotorInstance));

    inst->rx_id = config->rx_id;
    inst->motor_type = config->motor_type;
    inst->app_callback = config->app_callback;
    inst->app_id = config->app_id;

    s_motors[s_motor_cnt++] = inst;
    LOGINFO("[motor] Motor registered, rx_id=0x%03X", config->rx_id);
    return inst;
}
```
