## **基本描述**

BMI088 是一款用于检测6自由度（6DoF）运动和旋转的惯性测量单元（IMU）。它将两个惯性传感器的功能集成在一个设备中：一个先进的16位三轴陀螺仪和一个多功能、领先的16位三轴加速度计。

BMI088 旨在满足高性能消费类应用在恶劣振动环境（如无人机和机器人应用）中的所有要求。该IMU设计用于有效抑制由于PCB或整个系统结构的共振而偶尔发生的几百赫兹以上的振动。

该传感器具有高达 ±24g 的扩展测量范围，以避免在强信号暴露下发生信号削波。

一个评估电路（ASIC）转换微机电传感结构（MEMS）的输出，这些结构由博世工厂开发、生产和测试。相应的芯片组被封装在一个单一的LGA 3.0mm × 4.5mm × 0.95mm 外壳中。为了优化系统集成，BMI088配备了数字接口（SPI或I2C），提供从1.2V到3.6V的宽VDDIO电压范围。为了提供最大的性能和可靠性，每个设备都经过测试并已校准，可随时使用。

为了提高灵活性，陀螺仪和加速度计不仅可以单独操作，还可以为了数据同步的目的而连接在一起。片上功能包括用于加速度和陀螺仪数据的FIFO以及中断控制器。

BMI088 具有出色的温度特性，拥有极低的偏移温度系数（TCO）和灵敏度温度系数（TCS）。

**3. 快速入门指南 - 设备初始化**

为了正确初始化设备，应考虑以下步骤：

用户必须在硬件设计阶段就决定接口（I2C或SPI）：通过PS引脚，用户确定传感器应监听哪个接口（见第6章）。BMI088的陀螺仪部分根据PS引脚的选择初始化其I/O引脚。加速度计部分以I2C模式启动。它将保持在I2C模式，直到检测到CSB1引脚（加速度计的片选）上的上升沿，此时加速度计部分切换到SPI模式，并保持在该模式，直到下一次上电复位。要在初始化阶段将加速度计更改为SPI模式，用户可以执行一次虚拟SPI读取操作，例如读取寄存器ACC_CHIP_ID（获得的值将是无效的）。

上电复位后，陀螺仪处于正常模式，而加速度计处于挂起模式。要将加速度计切换到正常模式，用户必须执行以下步骤：

a. 给传感器上电
b. 等待 1 ms
c. 通过向 ACC_PWR_CTRL 写入 '4' 进入正常模式
d. 等待 450 微秒

### **第14页 / 共62页**

**4. 功能描述**

**4.1 电源管理和电源模式**

BMI088 有两个不同的电源引脚：

- VDD 是内部模块的主电源
- VDDIO 是一个独立的电源引脚，主要用于接口供电

只要每个引脚都在其工作范围内，两个引脚的电压水平彼此之间没有限制。此外，设备可以完全关闭（VDD = 0V），同时保持 VDDIO 供电（VDDIO > 0V），反之亦然。

当 VDDIO 供电关闭时，所有接口引脚（CSB, SDI, SCK, PS）必须保持接近 GNDio 电位。

该器件包含一个上电复位（POR）发生器。它在 VDD 和 VDDIO 上电后复位逻辑部分和寄存器值。这意味着，所有不等于默认设置（参考加速度计寄存器映射6.2和陀螺仪寄存器映射8.2）的应用特定设置，必须在 POR 后重新设置为其指定值。

请注意：POR 也会复位接口。对于陀螺仪部分，接口由 PS 引脚上的电压电平定义。加速度计部分的接口由 POR 发起时 CSB1 引脚的电压电平定义（见第3章）。

**4.1.1 电源模式：加速度计**

BMI088 加速度计的电源状态通过寄存器 ACC_PWR_CTRL 控制。寄存器 ACC_PWR_CTRL 用于启用/禁用加速度计和温度传感器。

要进入正常模式，必须将值 0x04 写入 ACC_PWR_CTRL。

要进入挂起模式，必须清除寄存器 ACC_PWR_CTRL。

**注意**：复位（POR 或软复位）后，传感器处于挂起模式，因此用户需要主动进入正常模式才能获取加速度值。

**注意**：POR 或软复位后，加速度传感器需要最多 1ms 的启动时间 (ts_up)。

**4.1.2 电源模式：陀螺仪**

陀螺仪有3种不同的电源模式。除了代表设备完全运行状态的正常模式外，还有两种节能模式：挂起模式和深度挂起模式。

上电后，陀螺仪处于正常模式，因此设备的所有部分都保持上电，并连续进行数据采集。

在挂起模式下，整个模拟部分断电。不执行数据采集。在挂起模式下，最新的角速率数据和所有配置寄存器的内容都会被保留。寄存器仍然可以读取（尽管它们不会被更新）。

通过向寄存器 GYRO_LPM1 写入 0x80 进入挂起模式。可以通过向 GYRO_LPM1 写入 0x00 或通过软复位（见 4.8）退出挂起模式。

尽管支持以全接口时钟速度（SCL 或 SCK）对寄存器进行写访问，但必须在两个连续的写周期之间插入一个等待周期（更多信息请参阅章节 9.2.1）。

在深度挂起模式下，设备达到最低可能的功耗。只有接口部分保持活动状态。不执行数据采集，配置寄存器的内容会丢失。

通过向寄存器 GYRO_LPM1 写入 0x20 进入深度挂起模式。可以通过向 GYRO_LPM1 写入 0x00 或通过软复位（见 4.8）退出深度挂起模式。

请注意，所有不等于默认设置的应用特定设置，在退出深度挂起模式后必须重新设置为其指定值。

**注意**：POR 或软复位后，或在不同的电源模式之间切换时，陀螺仪传感器需要最多 30ms 的时间才能达到新状态。在此期间应避免与传感器进行任何通信。

**4.2 传感器数据**

陀螺仪和加速度计传感器数据的宽度为 16 位（温度传感器为 11 位），以二进制补码表示。

每个轴的数据被拆分为一个 MSB 高字节和一个 LSB 低字节。读取传感器数据寄存器应始终从 LSB 部分开始。为了确保传感器数据的完整性，通过读取相应的 LSB 寄存器来锁定 MSB 寄存器的内容（影子过程）。

有关寄存器详情及如何解释这些寄存器中数据的详细信息，请参阅：

- 陀螺仪部分第 5.5.2 章
- 加速度计部分第 5.3.4 章
- 温度传感器第 5.3.7 章

突发访问机制提供了一种在 I2C 或 SPI 模式下高效读取角速率数据的方法。在突发访问期间，传感器在每字节后自动递增起始读取地址。突发访问允许在 I2C 总线上传输数据时，数据密度最多减少 50%。只要突发读取访问处于活动状态，所有读出寄存器中的传感器数据（角速率或加速度数据）都会被锁定。在突发读取访问模式下读取每个陀螺仪和加速度计部分的传感器数据寄存器，可确保所有读出寄存器中的传感器值属于同一采样点。

**4.6.1 加速度计**

自检功能允许通过向传感器核心施加静电力而不是外部加速度来检查传感器功能。通过物理偏转检测质量，测试了传感器的整个信号路径。激活自检会导致加速度数据中出现静态偏移。在自检期间施加到传感器的任何外部加速度或重力，都会在传感器输出中表现为加速度和自检信号的叠加。这意味着自检信号取决于传感器的方向。为了克服这个问题，完整的自检程序应在静态条件下执行，例如，当部件除了重力外未受到任何加速度激励时。

推荐的自检程序如下：

1. 通过向寄存器 ACC_RANGE (0x41) 写入 0x03 设置 ±24g 量程。
2. 通过向寄存器 ACC_CONF (0x40) 写入 0xA7 设置 ODR = 1.6kHz，连续采样模式，“正常模式”（norm_avg4）。
   连续滤波器功能：在 ACC_CONF 中设置第7位 "normal avg4 mode"：ACC_CONF = 0x02 << 4；ODR=1.6kHz：ACC_CONF = 0x0C
3. 等待 > 2ms
4. 使能正极性自检（即向寄存器 ACC_SELF_TEST (0x6D) 写入 0x0D）
5. 等待 > 50ms
6. 读取每个轴的加速度计偏移值（正自检响应）
7. 使能负极性自检（即向寄存器 ACC_SELF_TEST (0x6D) 写入 0x09）
8. 等待 > 50ms
9. 读取每个轴的加速度计偏移值（负自检响应）
10. 禁用自检（即向寄存器 ACC_SELF_TEST (0x6D) 写入 0x00）
11. 计算正负自检响应的差值，并与预期值（见下表）进行比较
12. 等待 > 50ms，让传感器稳定到正常模式稳态运行

**表 9：加速度计自检：正负自检信号之间的最小差值信号**

| X轴信号  | Y轴信号  | Z轴信号 |
| :------- | :------- | :------ |
| ≥1000 mg | ≥1000 mg | ≥500 mg |

建议在执行自检后对设备进行复位，因为自检响应也会影响中断产生。如果无法进行复位，则必须保持以下顺序以防止产生不需要的中断：禁用中断，更改中断参数，等待至少50ms，然后使能所需的中断。

**4.6.2 陀螺仪**

陀螺仪的内置自检设施不会偏转机械 MEMS 结构（如同加速度计自检那样），但此测试也提供了一种快速确定陀螺仪是否在指定条件下运行的方法。

要触发自检，必须设置地址 GYRO_SELF_TEST 的位 #0 (trig_bist)。当测试完成时，陀螺仪将设置位 #1 (bist_rdy)，然后可以在位 #2 (bist_fail) 中找到测试结果。'0' 表示测试无问题通过。如果发生故障，位 'bist_fail' 将被设置为 '1'。

可以通过读取地址 GYRO_SELF_TEST 的位 #4 来检查一个在后台连续运行的进一步测试。如果该位设置为 '1'，则表示传感器功能正常。

**4.7 新数据中断**

**4.7.2 陀螺仪**

陀螺仪提供一个新数据中断，每次在数据寄存器中存储新的 Z 轴角速率数据值后，将产生一个中断。中断在 280-400 μs 后自动清除。

**4.8 软复位**

软复位可以在任何时候启动：

- 对于加速度计部分，通过向寄存器 ACC_SOFTRESET（见 5.3.22）写入软复位命令（0xB6）
- 对于陀螺仪部分，通过向寄存器 GYRO_SOFTRESET（见 5.5.8）写入软复位命令（0xB6）

**4.9 FIFO**

BMI088 提供两个集成的 FIFO（先进先出）缓冲区，用于加速度计和陀螺仪传感器信号，帮助用户减少甚至省略时间关键的传感器读取访问，以获取高时间精度的数据。

**4.9.1 FIFO 操作模式**

FIFO 可以在不同模式下操作：FIFO（或称停止-满）模式和 STREAM 模式。

FIFO 或停止-满模式：在 FIFO 或停止-满模式下，传感器值被依次存储在 FIFO 缓冲区中，直到缓冲区满。
STREAM 模式：STREAM 模式的工作方式类似于 FIFO 模式，区别在于一旦缓冲区满，FIFO 中最旧的数据将被传感器的最新数据覆盖。

**4.9.2 FIFO 中断**

FIFO 缓冲区支持两种类型的中断：

水印中断：当 FIFO 缓冲区的填充水平达到用户定义的阈值时触发。
FIFO 满中断：当 FIFO 满时触发。

**4.9.3 加速度计传感器 FIFO 缓冲区**

BMI088 的加速度计部分有一个集成的 1024 字节数据 FIFO。FIFO 以帧为单位从数据寄存器捕获数据，每一帧仅包含一个传感器样本。

**4.9.3.1 启用 FIFO 和选择模式**

加速度计传感器数据的 FIFO 通过设置寄存器 0x49（见 5.3.15）的位 #6 来启用。

**4.9.3.1.1 模式选择**

当需要 STREAM 模式时，必须清除寄存器 0x48 的位 #0（设置为 '0'）（上电复位时的默认值，见 5.3.14）。
对于 FIFO 或停止-满模式，必须在寄存器 0x48 中将位 #0 设置为 '1'。

**4.9.3.1.2 FIFO 数据采样率**

FIFO 的输入数据率与配置的传感器 ODR 相同。但是，可以通过选择 2^k（k = [0,1,...,7]）的降采样因子来降低。因子 k 必须写入寄存器 0x45 的位 #4-6（见 5.3.12）。

**4.9.3.1.3 加速度计的 FIFO 与外部中断同步（标签应用）**
如果 INT1 和/或 INT2 引脚被配置为输入引脚（通过在寄存器 INT2_IO_CTRL 中设置 int2_io 和/或在寄存器 INT1_IO_CTRL 中设置 int1_io），这些引脚上的信号也可以记录在 FIFO 中，并且帧会被相应地“标记”。因此，需要在寄存器 FIFO_CONFIG_1（见 5.3.15）中激活这些引脚以进行 FIFO 记录。

**4.9.3.2.6 FIFO 部分帧读取和开销**

当一帧仅通过寄存器 FIFO_DATA 被部分（不完全）读取时，它将在下一次访问时被完整重复。如果在第一次部分读取和第二次读取尝试之间发生 FIFO 溢出，该帧可能会被删除。

当从 FIFO 读取的数据多于其包含的有效数据时，返回 0x8000。

**4.9.3.3 FIFO 中断**

FIFO 支持两种中断，FIFO 满中断和水印中断：

FIFO 满中断在 FIFO 填充水平超过满阈值时发出。满阈值在最后两帧存储到 FIFO 之前达到。
FIFO 水印中断在 FIFO 填充水平大于或等于寄存器 FIFO_WTM（见 5.3.13）中定义的水印水平时发出。

为了启用/使用 FIFO 满中断或水印中断，需要通过 INT1_INT2_MAP_DATA（见 5.3.18）将它们映射到所需的中断引脚。

当对寄存器 FIFO_DATA 的读取操作正在进行时，这两种中断都会被抑制。锁存的 FIFO 中断只有在读取状态寄存器且填充水平低于相应的 FIFO 中断（满或水印）时才会被清除。

**4.9.4.2 FIFO 数据读出**

FIFO 存储的数据也可在读出寄存器 0x02-0x07 中获得。因此，所有配置设置都适用于 FIFO 数据和数据读出寄存器。FIFO 读出可以通过寄存器 0x3F (FIFO_DATA) 进行。可以使用突发模式进行读出。一次突发可以读出一个或多个帧。如果由于读取操作不完整而未完全读取一帧，则该帧的剩余部分将丢失。在这种情况下，FIFO 会在下一次读取操作时对齐到下一帧。

数据格式见 5.5.17。

**4.9.4.2.1 使用陀螺仪 FIFO 的接口速度要求**
为了有效使用 FIFO，需要快速读取大量数据块。根据传感器的输出数据速率，这可能对接口提出要求。

陀螺仪的输出数据速率由滤波器配置决定（参见传感器数据手册）。所需的接口速度取决于所选速率。

对于 400kHz 的 I2C 速度，可以使用任何滤波器模式。对于 200kHz 的 I2C 速度，建议仅使用输出数据速率为 1kHz 及以下的模式。对于 100kHz 的 I2C 速度，建议仅使用输出数据速率为 400Hz 及以下的模式。

**4.9.4.3 FIFO 帧计数器和溢出标志**

帧计数器（地址 0x0E 位 <6:0>，见 5.5.4）指示缓冲区的当前填充水平。如果尽管 FIFO 已满但仍将额外的帧写入缓冲区，则会设置溢出标志（寄存器 0x0E 位 7）。如果 FIFO 被复位，则 FIFO_CONFIG_1 寄存器中指示的 FIFO 填充水平帧计数器 <6:0> 被设置为 '0'，并且每次对 FIFO 配置寄存器进行写操作时，溢出标志都会被复位。

注意：当 FIFO 填充水平帧计数器 <6:0> 由于从 FIFO_DATA 寄存器读取而递减到 '0' 时，溢出位不会被复位，只有当对 FIFO 配置寄存器执行写操作时才会被复位。

**4.9.4.4 FIFO 中断**

FIFO 支持两种中断，FIFO 满中断和水印中断：

FIFO 满中断在缓冲区完全填满样本时发出。在 FIFO 模式下，这发生在存储了 100 个样本之后，在 STREAM 模式下，发生在先前为空的 FIFO 中存储了 99 个样本之后。FIFO 满中断的状态可以通过 INT_STATUS_1 寄存器 0x0A 中的状态位读回。
水印中断在缓冲区中的填充水平达到寄存器 0x3D 中定义的水印水平触发值时发出。水印的状态可以通过地址 0x0A 的位 4 (fifo_int) 状态位读回。向寄存器 0x3D 中的水印水平触发值写入数据会清除 FIFO 缓冲区。

**5.3.4 寄存器 0x12-0x17：ACC 数据**

包含加速度传感器输出的寄存器。传感器输出以带符号的16位数形式存储，采用二进制补码格式，每个轴占用2个寄存器。根据这些寄存器，可以如下计算加速度值：

Accel*X_int16 = ACC_X_MSB * 256 + ACC*X_LSB
Accel_Y_int16 = ACC_Y_MSB * 256 + ACC_Y_LSB
Accel_Z_int16 = ACC_Z_MSB \* 256 + ACC_Z_LSB

当读取包含某个加速度值LSB的寄存器时，相应的MSB寄存器会在内部被锁定，直到其被读取。通过这种机制，可以确保LSB和MSB值属于同一个加速度值，并且不会在单独寄存器的读取间隔之间被更新。

单位是LSB。从LSB到加速度（mg）的转换基于量程设置，可以如下计算（<0x41>：ACC*RANGE寄存器的内容）：
Accel_X_in_mg = Accel_X_int16 / 32768 * 1000 _ 2^(<0x41>+1) _ 1.5
Accel*Y_in_mg = Accel_Y_int16 / 32768 * 1000 _ 2^(<0x41>+1) _ 1.5
Accel*Z_in_mg = Accel_Z_int16 / 32768 * 1000 \_ 2^(<0x41>+1) \* 1.5

**5.5.2 寄存器 0x02 - 0x07：角速率数据**

包含角速度传感器输出的寄存器。传感器输出以带符号的16位数形式存储，采用二进制补码格式，每个轴占用2个寄存器。根据这些寄存器，可以如下计算陀螺仪值：

Rate*X: RATE_X_MSB * 256 + RATE*X_LSB
Rate_Y: RATE_Y_MSB * 256 + RATE_Y_LSB
Rate_Z: RATE_Z_MSB \* 256 + RATE_Z_LSB

当读取包含某个速率值LSB的寄存器时，相应的MSB寄存器会在内部被锁定，直到其被读取。通过这种机制，可以确保LSB和MSB值属于同一个速率值，并且不会在单独寄存器的读取间隔之间被更新。

单位是LSB。从LSB到角速度（度每秒）的转换基于量程设置（见5.5.5）。例如，对于寄存器0x0F中的默认量程设置0x00，适用以下转换表：

| 传感器输出 [LSB] | 角速率 (在 2000°/s 量程模式下) |
| :--------------- | :----------------------------- |
| +32767           | +2000°/s                       |
| ...              | ...                            |
| 0                | 0°/s                           |
| ...              | ...                            |
| -32767           | -2000°/s                       |

**6. 数字接口**
BMI088 支持两种串行数字接口协议，用于与主机设备进行从设备通信：SPI 和 I²C。活动接口通过引脚 #07 (PS) '协议选择' 引脚的状态选择：

- PS = 'VDDIO' 选择 I²C
- PS = 'GND' 选择 SPI

**重要提示：**

- 请注意，对于 SPI 协议，BMI088 加速度计部分的初始化过程需要一些额外的步骤（见第3章）。
- 另请注意，由于封装的引脚在加速度计和陀螺仪部分之间共享，建议不要为这两个部分配置不同的接口。

两种数字接口部分共享相同的引脚。此外，每个惯性传感器（加速度计和陀螺仪）提供特定的接口引脚，允许用户独立操作每个惯性传感器。每个接口和每个惯性传感器的映射如下表所示：

**6.1.2 加速度计部分的 SPI 接口**

在加速度计部分的读操作情况下，请求的数据不会立即发送，而是先发送一个虚拟字节，在此虚拟字节之后才传输实际的寄存器内容。

数据位使用如下：

位 #0：读/写位。为 0 时，数据 SDI 写入芯片。为 1 时，从芯片读取数据 SDO。
位 #1-7：地址 AD(6:0)。
位 #8-15：在写模式下，这些是 SDI 数据，将写入地址。在读模式下，这些位包含不可预测的值，用户必须读取位 #16-23 才能从读取地址获得实际数据。

I²C 协议工作原理如下：

起始（START）：总线上的数据传输以 SDA 线在 SCL 为高电平时从高到低的跳变开始（由 I²C 总线主设备发出的起始条件 (S)）。一旦主设备传输了起始信号，总线即被视为忙碌。

停止（STOP）：每次数据传输应由主设备生成的停止信号 (P) 终止。停止条件是 SDA 线在 SCL 为高电平时从低到高的跳变。

应答（ACK）：传输的每个字节数据都必须被应答。它由接收器发送的一个应答位指示。在应答脉冲期间，发送器必须释放 SDA 线（不拉低），而接收器则必须将 SDA 线拉低，以便在应答时钟周期的高电平期间保持稳定低电平。

在以下图表中，使用这些缩写：
S 起始
P 停止
ACKS 从设备应答
ACKM 主设备应答
NACKM 主设备非应答
RW 读/写

不支持起始后立即跟停止（没有 SCL 从 'VDIO' 到 'GND' 的切换）。如果出现这种组合，设备将无法识别停止。

**I2C 写访问：**

I2C 写访问可用于在一个序列中写入一个数据字节。

该序列从主设备生成的起始条件开始，后跟 7 位从设备地址和一个写位 (RW = 0)。从设备发送一个应答位 (ACK = 0) 并释放总线。然后主设备发送一个字节的寄存器地址。从设备再次确认传输，并等待 8 位数据，这些数据将被写入指定的寄存器地址。在从设备确认数据字节后，主设备生成停止信号并终止写入协议。

加速度计 I2C 写访问示例，将 0xA8 写入地址 0x40（即设置连续滤波器功能，平均 4 个样本，ODR 为 100Hz）：

<center>**图 4：I2C 写**</center>

**I2C 读访问：**

I2C 读访问也可用于在一个序列中读取一个或多个数据字节。

一个读取序列由一个单字节 I2C 写阶段后跟一个 I2C 读阶段组成。传输的两部分必须由重复起始条件 (Sr) 分隔。I2C 写阶段寻址从设备并发送要读取的寄存器地址。在从设备确认传输后，主设备再次生成起始条件，并发送从设备地址和一个读位 (RW = 1)。然后主设备释放总线，并等待从设备读出的数据字节。在每个数据字节之后，主设备必须生成一个应答位 (ACK = 0) 以启用进一步的数据传输。来自主设备的非应答 (NACK) 会停止从设备传输数据。从设备释放总线，以便主设备生成停止条件并终止传输。寄存器地址自动递增，因此可以顺序读出多个字节。一旦新的数据读取传输开始，起始地址将设置为最新 I2C 写命令中指定的寄存器地址。默认情况下，起始地址设置为 0x00。通过这种方式，可以从相同的起始地址重复进行多字节读取。

**8.6 操作说明**

微机械传感器旨在高精度地感测加速度，即使在低幅度下也是如此，并且在传感器元件内部包含高度敏感的结构。MEMS 传感器可以承受高达数千 g 的机械冲击。但是，在极端冲击负载条件下（例如，用锤子敲击传感器或其附近、将传感器掉落到坚硬表面等），这些限制可能会被超过。

我们建议在规定的限制范围内，在明确且合格的安装过程中，避免在运输、处理和安装传感器期间产生超过规定限值的 g 力。

该器件内置了防止高静电放电或电场的保护措施（例如 2kV HBM）；但是，与任何其他 CMOS 组件一样，应采取防静电措施。除非另有规定，只有当所有端子电压保持在电源电压范围内时，才能正常运行。未使用的输入必须始终连接到确定的逻辑电压电平。
