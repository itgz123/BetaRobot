  ---
  辨识方案总结

  根据我读取的文件，整个项目经历了 V1 → V2 → V3 (当前) 三个版本的演进：

  ---
  V1 版本 (old/v1.py) - 时域最小二乘为主

  核心方法：
  1. 频域惯量辨识 — 曲线拟合法，|H(ω)| = 1/√((Jω)² + cv²)
  2. 时域批量最小二乘法 — 一次性求解 J + 双向摩擦 + 重力残差

  问题：
  - R² ≈ 0.18（拟合优度极低）
  - 残差 (0.016 Nm) >> 摩擦信号 (0.0005 Nm)
  - np.gradient() 数值微分放大噪声，摩擦参数被噪声淹没

  ---
  V2 版本 (old/v2.py) - 频域惯量 + 多窗口精炼

  核心改进：
  1. 频域惯量 (V1 式) — 保持 V1 的 curve_fit 法
  2. SG 滤波粗摩擦提取 — 用 Savitzky-Golay 微分替代 np.gradient
  3. 多窗口短时仿真精炼 — 5s 窗口 (50% 重叠) 优化摩擦参数
    - 固定 J，仅优化 4 个摩擦参数
    - 窗口间标准差衡量参数可靠性

  问题：
  - 5 参数同时优化 (J + cv± + Tc±) 导致 J 被摩擦污染，高估 4 倍
  - 60s 积分漂移严重

  ---
  V3 版本 (当前 identify_axis_mit_lite.py) - 综合最优方案

  核心洞见（来自大量实验验证）：

  ▎ J 必须来自 V1 curve_fit (频域)，5 参数同时优化会导致 J 高估 4 倍。固定 J 后 4 参数优化收敛性大幅提升。

  ---
  性能优化要点

  核心问题诊断

  log.txt 中明确指出：

  ▎ 问题不在算法，而在数据。时域最小二乘法本身数学上完美，但扫频信号下摩擦项需要经过 T_friction = T_chirp - J * accel 来分离，而 np.gradient()
  ▎ 数值微分放大了噪声，使摩擦信号完全被淹没。

  频域法表现好是因为 FFT 的积分平均效应天然抑制了噪声。

  关键代码改进

  1. SG 滤波微分替代 np.gradient：

  accel = np.gradient(speed, dt)

  # V2/V3 (SG 滤波微分):
  accel = savgol_filter(speed, window_length=15, polyorder=3, deriv=1, delta=dt)
  2. 固定 J 的多窗口优化：
  # V3 核心改进: 固定 J (来自频域), 仅优化 4 个摩擦参数
  def objective(x):
      cp, cn, Tcp, Tcn = abs(x[0]), abs(x[1]), abs(x[2]), abs(x[3])
      _, om_p = self._simulate(th0, om0, tm, dt, J, cp, cn, Tcp, Tcn, N)
      return np.sqrt(np.mean((om_p - sm) ** 2))  # 速度 RMSE

  alpha_max = 1e5  # rad/s², 防输出溢出

  总结

  V3 (当前代码) 是综合最优方案：
  - 惯量 J：频域 curve_fit（实验验证最可靠）
  - 摩擦参数：固定 J + 多窗口仿真精炼（避免参数模糊）
  - 验证：NRMSE + SNR + 窗口一致性（全面评估）

  核心洞见：J 和摩擦必须分开辨识，5 参数同时优化会导致 J 被摩擦污染。
