# 多正弦叠加参数辨识

本文档介绍 `AXIS_LITE_STAGE_IDENTIFY_OLS` 阶段的多正弦叠加电流设定值生成及时域正交可分离最小二乘法辨识原理。

---

## 一、时域正交可分离最小二乘法

### 1.1 基本原理

对于单轴关节的动力学方程：

$$J\ddot{\theta} + c_v\dot{\theta} + T_c \cdot \text{sign}(\dot{\theta}) = \tau$$

其中：
- $J$ — 转动惯量 (kg·m²)
- $c_v$ — 粘性摩擦系数 (Nm·s/rad)
- $T_c$ — 库仑摩擦 (Nm)
- $\tau$ — 输入力矩 (Nm)

**核心思想**：使用多正弦叠加信号作为激励，利用正弦函数的正交性，在时域内分离并辨识各参数。

### 1.2 正弦信号的正交性

对于两个不同频率的正弦函数：

$$\int_0^T \sin(\omega_1 t) \cdot \sin(\omega_2 t) \, dt = 0 \quad (\text{当 } \omega_1 \neq \omega_2)$$

这个性质使得我们可以：

1. **分离各频率成分的贡献**
2. **独立辨识各参数**
3. **避免参数耦合导致的辨识误差**

## 二、正弦波频率选择要求

### 2.1 基本要求

| 要求         | 说明                               | 数学表达                                                                      |
| ------------ | ---------------------------------- | ----------------------------------------------------------------------------- |
| **正交性**   | 各频率在观测时长内正交             | $\omega_i \neq \omega_j$ 且 $\int_0^T \sin(\omega_i t)\sin(\omega_j t)dt = 0$ |
| **整周期**   | 每个频率在观测时长内完成整数个周期 | $T = n_i \cdot \frac{2\pi}{\omega_i}, \quad n_i \in \mathbb{Z}^+$             |
| **频带覆盖** | 频率范围覆盖系统主要动态特性       | 通常 0.5~20 Hz                                                                |
| **频率间隔** | 相邻频率差异足够大                 | $\|\omega_{i+1} - \omega_i\| > \Delta\omega_{min}$                            |

### 2.2 整周期条件详解

**最重要要求**：观测时长 $T$ 必须是每个正弦波周期的整数倍。

$$T = n_1 \cdot T_1 = n_2 \cdot T_2 = \cdots = n_N \cdot T_N$$

其中 $T_i = \frac{1}{f_i}$ 是第 $i$ 个正弦波的周期，$n_i$ 是正整数。

**实现方法**：

1. 选择基准周期 $T_{base}$
2. 所有频率满足：$f_i = \frac{n_i}{T_{base}}$，即频率之比为整数比

**示例**（观测时长 $T = 10\text{s}$）：

| 频率 (Hz) | 周期 (s) | 10s内周期数 | 是否整周期 |
| --------- | -------- | ----------- | ---------- |
| 1.0       | 1.0      | 10          | ✓          |
| 2.0       | 0.5      | 20          | ✓          |
| 4.0       | 0.25     | 40          | ✓          |
| 5.0       | 0.2      | 50          | ✓          |
| **1.5**   | 0.667    | **15**      | ✓          |
| **1.7**   | 0.588    | **17**      | ✓          |

### 2.3 频率选择策略

#### 策略一：倍频法（推荐）

频率为基准频率的整数倍：

```c
float f_base = 1.0f;  // 基准频率 1 Hz
float freqs[] = {1.0f, 2.0f, 4.0f, 8.0f};  // 1×, 2×, 4×, 8×
```

**优点**：自然满足整周期条件，正交性最好

#### 策略二：谐波相关法

频率为基准频率的谐波：

```c
float f_base = 0.5f;
float freqs[] = {0.5f, 1.0f, 1.5f, 2.0f, 2.5f, 3.0f};  // f_base 的 1,2,3,4,5,6 倍
```

#### 策略三：对数间隔法

频率在对数尺度上均匀分布：

```c
// 0.5~20 Hz，对数间隔
float freqs[] = {0.5f, 1.0f, 2.0f, 4.0f, 8.0f, 16.0f};
```

**注意**：需确保整周期条件，调整时长 $T$

### 2.4 频率数量建议

| 频率数量 | 适用场景           | 说明                        |
| -------- | ------------------ | --------------------------- |
| 2-3      | 简单模型（仅惯量） | 快速辨识                    |
| 4-6      | 常规辨识           | 平衡精度与时间              |
| 7-8      | 高精度辨识         | 最多支持 8 个（宏定义限制） |

## 三、振幅选择

### 3.1 基本原则

1. **信噪比**：振幅足够大，保证响应信号的信噪比
2. **安全性**：振幅不超过电机和机械结构的承受能力
3. **线性区间**：确保系统工作在线性范围内

### 3.2 振幅分配策略

#### 等幅分配

```c
float amplitudes[] = {0.01f, 0.01f, 0.01f, 0.01f};  // 各频率振幅相同
```

**优点**：简单，各频率贡献均匀

#### 频率补偿分配

低频振幅较大，高频振幅较小：

```c
// 低频大振幅，高频小振幅
float amplitudes[] = {0.02f, 0.015f, 0.01f, 0.005f};
```

**原理**：惯量辨识对高频更敏感，摩擦辨识对低频更敏感

### 3.3 振幅参考值

| 电机类型 | 典型振幅范围 (Nm) | 说明     |
| -------- | ----------------- | -------- |
| M2006    | 0.01 ~ 0.1        | 小惯量   |
| M3508    | 0.05 ~ 0.3        | 中等惯量 |
| M6020    | 0.1 ~ 0.5         | 大惯量   |

## 四、实现细节

### 4.1 多正弦信号生成

```c
/**
 * @brief 生成多正弦叠加力矩设定值
 * @param params 多正弦参数结构体指针
 * @param t 当前时间 (s)
 * @return 叠加力矩 (Nm)
 *
 * 信号形式：τ(t) = Σ A_i * sin(2π * f_i * t)
 */
static inline float GenerateMultiSineTorque(MultiSineParam_s *params, float t)
{
    float torque = 0.0f;

    for (uint8_t i = 0; i < params->num_freqs; i++)
    {
        float w = M_2PI * params->freqs[i];
        float phase = w * t;
        torque += params->amplitudes[i] * BSP_Math_Sin(phase);
    }

    return torque;
}
```

### 4.2 使用示例

```c
// 配置多正弦辨识阶段
AxisMitLite_Init_Config_s cfg = {
    .motor = motor_instance,
    .stage = AXIS_LITE_STAGE_IDENTIFY_OLS,
    .delay_ms = 1000,

    // 多正弦叠加参数
    .multi_sine_params = {
        // 频率：倍频法，满足整周期条件
        .freqs = {1.0f, 2.0f, 4.0f, 8.0f},
        // 振幅：低频大、高频小
        .amplitudes = {0.02f, 0.015f, 0.01f, 0.005f},
        .num_freqs = 4,
        .duration = 10.0f,  // 10s，是所有频率的整数周期
    },

    // ... 其他参数
};

AxisMitLiteInstance inst;
AxisMitLiteInit(&inst, &cfg);
```

## 五、性能分析

### 5.1 `arm_sin_f32` 执行时间

`arm_sin_f32` 使用 **256点查表 + 线性插值** 实现，执行效率极高。

| 平台                    | CPU频率 | 估算周期数 | 单次调用时间 | 8次调用时间 |
| ----------------------- | ------- | ---------- | ------------ | ----------- |
| **STM32F407 (DJI_C)**   | 168 MHz | ~20-30     | ~150 ns      | ~1.2 μs     |
| **STM32F427 (DJI_A)**   | 180 MHz | ~20-30     | ~140 ns      | ~1.1 μs     |
| **STM32H723 (DM_MC02)** | 550 MHz | ~15-20     | ~30 ns       | ~0.24 μs    |

### 5.2 算法开销分析

`AXIS_LITE_STAGE_IDENTIFY_OLS` 阶段的主要计算开销：

| 操作                                | 单次耗时 (DJI_C) | 单次耗时 (DM_MC02) |
| ----------------------------------- | ---------------- | ------------------ |
| `CalcTimeSinceDelay()`              | ~50 ns           | ~10 ns             |
| `fmodf()`                           | ~200 ns          | ~40 ns             |
| `GenerateMultiSineTorque()` (4频率) | ~600 ns          | ~120 ns            |

| **总计 (4频率)**                    | **~950 ns**      | **~190 ns**        |
| **总计 (8频率)**                    | **~1.55 μs**     | **~310 ns**        |

### 5.3 实时性评估

| 控制周期        | 可用时间 | 算法占用 (8频率, DJI_C) | 占用率    |
| --------------- | -------- | ----------------------- | --------- |
| 1 kHz (1 ms)    | 1000 μs  | ~1.5 μs                 | **0.15%** |
| 500 Hz (2 ms)   | 2000 μs  | ~1.5 μs                 | **0.08%** |
| 10 kHz (0.1 ms) | 100 μs   | ~1.5 μs                 | **1.5%**  |

**结论**：即使在 10 kHz 控制频率下，算法开销也仅占 1.5%，完全满足实时性要求。

### 5.4 与标准库 `sinf()` 对比

| 函数          | 实现方式        | 周期数 (估算) | 精度       |
| ------------- | --------------- | ------------- | ---------- |
| `arm_sin_f32` | 查表 + 线性插值 | 20-30         | ~0.001 rad |
| `sinf()`      | 泰勒级数/多项式 | 50-100        | ≤1 ULP     |

**选择建议**：
- 控制应用：使用 `arm_sin_f32`（`BSP_Math_Sin`），速度优先
- 需要高精度时：使用 `sinf()`

## 六、辨识数据采集

### 6.1 需要采集的数据

在进行离线辨识时，需要记录以下数据：

| 通道 | 数据     | 单位  | 用途           |
| ---- | -------- | ----- | -------------- |
| CH1  | 反馈位置 | rad   | 位置响应       |
| CH2  | 反馈速度 | rad/s | 速度响应       |
| CH3  | 电机电流 | A     | 实际力矩       |
| CH9  | 摩擦前馈 | Nm    | 输入力矩设定值 |

### 6.2 数据长度要求

为确保正交性，建议采集时长满足：

$$T_{acquire} \geq T_{duration} \times 2$$

即至少采集 2 个完整周期，用于验证辨识结果的一致性。

## 七、常见问题

### Q1: 频率不满足整周期条件会怎样？

**影响**：
- 频谱泄漏
- 正交性下降
- 辨识精度降低

**解决**：调整频率或时长，确保整周期条件

### Q2: 频率数量过多会怎样？

**影响**：
- 单个频率能量下降
- 信噪比降低
- 辨识发散风险增加

**建议**：4-6 个频率通常足够

### Q3: 如何验证频率选择的正确性？

**方法**：
1. 检查各频率在 $T$ 内是否为整数周期
2. 计算相关矩阵的条件数
3. 使用仿真数据验证辨识精度
