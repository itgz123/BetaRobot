### 实例结构体参数读取直接读结构体，写参数必须用函数
1. 避免变量意外修改
2. 读取方便，高效
### DRV/BSP 所有功能都要可以通过宏开启或关闭
1. 减少 Flash/RAM 占用
### 检查 `const`、`extern`、`register`、`static`、`volatile`
1. 不需要修改的数表用const
2. 不能修改的结构体成员（例如*_INSTANCE_DEF宏定义的协议缓冲区长度）用const
3. 尽量不用extern，除非导出不可修改的数表
4. 中断会被修改，需要每次cpu从内存读取的变量用volatile
5. 本地变量用static，app层全部都要用static
### register，init，config
1. register不可以重入，会改变static静态变量
2. config可以重入，修改register过的参数
3. 不需要register的地方直接init
4. 务必根据每个drv或者bsp的实际情况严格检查cconfig参数
### 检查每个 DRV/BSP 的所有 static 函数，不要每个 DRV 自己定义一个 clamp 限幅函数
1. 节约flash
2. 提高可读性
### app文件中循环运行的函数的变量全部使用static文件域的全局变量
1. 调试只能查看全局变量
2. 可读性高，避免突然出现一个变量
3. 变量放在静态区，而不是freertos任务栈

### 接收信息的drv(sbus,vt03,bmi088,motor)是否需要时间戳
1. 除了sbus,vt03来自人的信号可以不需要时间戳，因为人的操作不够精准，不要也没关系
2. 来自bmi088,motor的数据需要精确时间戳，用来姿态观测等需要精确数据
### 是否需要drv内部计算时间
1. pid等控制器的dt使用控制时间间隔，可以内部集成dwt计算时间
2. mahony,kalman滤波需要数据时间差，根据数据帧时间戳从外部传入
