# 内存与存储原理

本文档详细介绍嵌入式系统的内存模型，分为理想模型和实际模型两部分。

---

# 第一章：理想模型 - RAM与Flash运行原理

## 1.1 物理特性对比

| 特性         | Flash                | RAM            |
| ------------ | -------------------- | -------------- |
| **用途**     | 存储程序代码和常量   | 存储运行时数据 |
| **掉电后**   | 数据保留             | 数据丢失       |
| **速度**     | 较慢（需要等待状态） | 快             |
| **写入方式** | 需要擦除后才能写     | 随机读写       |
| **生命周期** | 有限（约10万次擦写） | 无限           |
| **成本**     | 低（每字节）         | 高（每字节）   |

## 1.2 内存区域划分

### 1.2.1 整体布局

```
高地址
┌─────────────────────┐
│      栈 (Stack)     │ ← 向下增长（局部变量、函数调用）
├─────────────────────┤
│         ↓           │
│                     │
│         ↑           │
├─────────────────────┤
│      堆 (Heap)      │ ← 向上增长（malloc/free）
├─────────────────────┤
│    .bss段 (未初始化) │ ← 全局/静态变量，初始化为0
├─────────────────────┤
│    .data段 (已初始化)│ ← 全局/静态变量，有初始值
├─────────────────────┤
│   Flash中的镜像      │ ← .data的初始值存储在这里
├─────────────────────┤
│    .rodata段 (只读)  │ ← const常量，字符串字面量
├─────────────────────┤
│    .text段 (代码)    │ ← 程序代码
└─────────────────────┘
低地址
```

### 1.2.2 各区域详解

#### .text段（代码段）

- **位置**：Flash
- **内容**：程序的可执行机器码
- **特点**：只读，大小固定

```c
// 所有函数代码都存放在.text段
void my_function(void) {
    // ...
}

int main(void) {
    // ...
    return 0;
}
```

#### .rodata段（只读数据段）

- **位置**：Flash
- **内容**：常量、字符串字面量、const全局变量
- **特点**：只读，不占用RAM

```c
const int config_value = 100;           // 全局const变量 → .rodata
const char* error_msg = "Error";        // 字符串字面量 → .rodata
const uint8_t lookup_table[256] = {...};// 查找表 → .rodata

// 注意区别
char* str = "Hello";    // "Hello"在.rodata，str指针在RAM
char str2[] = "Hello";  // 数组内容在RAM（从.rodata复制）
```

#### .data段（已初始化数据段）

- **位置**：RAM（运行时），Flash（存储初始值）
- **内容**：有初始值的全局变量、静态变量
- **特点**：占用Flash+RAM双份空间

```c
int g_initialized = 100;            // .data（占Flash存储初始值 + RAM运行空间）
static int s_counter = 0;           // .data
float g_array[10] = {1,2,3};        // .data

// 启动过程：
// 1. Flash中存储初始值 [100, 0, 1,2,3,...]
// 2. 启动代码将初始值复制到RAM
// 3. 程序运行时在RAM中读写
```

#### .bss段（未初始化数据段）

- **位置**：RAM
- **内容**：未初始化或初始化为0的全局变量、静态变量
- **特点**：只占RAM，不占Flash空间

```c
int g_buffer[256];              // .bss（只占RAM）
static int s_flag;              // .bss
double g_matrix[10][10];        // .bss

// 启动过程：
// 1. 不占用Flash空间（没有初始值需要存储）
// 2. 启动代码直接在RAM中清零
// 3. 程序运行时在RAM中读写
```

#### 栈（Stack）

- **位置**：RAM
- **内容**：局部变量、函数参数、返回地址、寄存器保存
- **特点**：自动管理，向低地址增长

```c
void function(int param)         // param在栈上
{
    int local_var = 10;          // 局部变量 → 栈
    char buffer[64];             // 局部数组 → 栈

    if (condition) {
        int block_var;           // 块作用域变量 → 栈
    }
}   // 函数返回后，栈上所有局部变量自动释放
```

#### 堆（Heap）

- **位置**：RAM
- **内容**：动态分配的内存
- **特点**：手动管理，向高地址增长

```c
void example(void) {
    int* ptr1 = malloc(100);     // 动态分配 → 堆
    char* ptr2 = calloc(50, 1);  // 动态分配并清零 → 堆

    free(ptr1);                  // 手动释放
    free(ptr2);
}
```

## 1.3 C语言语法与内存区域对应

### 1.3.1 速查表

| C语言定义               | 存储位置 | 段名    | 占用Flash   | 占用RAM   |
| ----------------------- | -------- | ------- | ----------- | --------- |
| `int g = 1;`            | RAM      | .data   | ✓（初始值） | ✓         |
| `int g;`                | RAM      | .bss    | ✗           | ✓         |
| `const int g = 1;`      | Flash    | .rodata | ✓           | ✗         |
| `static int s = 1;`     | RAM      | .data   | ✓（初始值） | ✓         |
| `static int s;`         | RAM      | .bss    | ✗           | ✓         |
| `void func(){ int x; }` | 栈       | -       | ✗           | ✓（临时） |
| `malloc(100)`           | 堆       | -       | ✗           | ✓         |
| `"string"`              | Flash    | .rodata | ✓           | ✗         |
| `char arr[] = "str";`   | RAM      | .data   | ✓           | ✓         |

### 1.3.2 关键字详解

#### static关键字

```c
// 全局作用域的static：限制链接可见性
static int module_private = 10;  // 只在本文件可见，存放在.data

// 函数内的static：持久存储
void counter(void) {
    static int count = 0;        // 存放在.data，程序生命周期
    count++;                     // 值在多次调用间保持
}

// 静态局部变量 vs 普通局部变量
void example(void) {
    int normal = 0;              // 栈上，每次调用重新初始化
    static int persistent = 0;   // .data，只初始化一次
}
```

#### const关键字

```c
// const全局变量 → .rodata（Flash）
const int MAX_VALUE = 100;

// const局部变量 → 栈（仍是栈上的只读变量）
void func(void) {
    const int local_const = 10;  // 栈上
}

// const指针的不同含义
const int* p1;        // 指向的内容不可变
int* const p2;        // 指针本身不可变
const int* const p3;  // 都不可变
```

#### volatile关键字

```c
// volatile不改变存储位置，只告诉编译器不要优化访问
volatile int hardware_reg;       // 可能是内存映射寄存器
volatile uint32_t* DMA_ADDR;     // DMA可能修改的地址

// 常见使用场景
#define REG_BASE    0x40000000
#define GPIO_ODR    (*(volatile uint32_t*)(REG_BASE + 0x14))
```

## 1.4 程序启动初始化流程

### 1.4.1 启动流程图

```
复位
    │
    ▼
┌─────────────────┐
│ 1. 硬件初始化    │  ← 设置栈指针(SP)到RAM末尾
│    (启动代码)    │  ← 设置PC到Reset_Handler
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 2. .data段初始化 │  ← 从Flash复制初始值到RAM
│                 │  memcpy(_sdata, _sidata, size)
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 3. .bss段清零   │  ← 将未初始化数据区清零
│                 │  memset(_sbss, 0, size)
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 4. 系统初始化    │  ← SystemInit()：时钟配置等
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 5. C运行环境    │  ← C库初始化（如有）
│    初始化       │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 6. main()       │  ← 进入用户程序
└─────────────────┘
```

### 1.4.2 启动代码示例

```c
// 典型的启动文件逻辑（简化版）
void Reset_Handler(void) {
    // 1. 复制.data段从Flash到RAM
    uint32_t *src = &_sidata;   // Flash中的数据
    uint32_t *dst = &_sdata;    // RAM目标地址
    while (dst < &_edata) {
        *dst++ = *src++;
    }

    // 2. 清零.bss段
    dst = &_sbss;
    while (dst < &_ebss) {
        *dst++ = 0;
    }

    // 3. 调用系统初始化
    SystemInit();

    // 4. 调用main
    main();

    // 5. 不应该返回
    while(1);
}
```

# 第二章：实际模型 - STM32 F4/H7内存架构

## 2.1 STM32F407内存架构

### 2.1.1 内存映射

```
地址空间布局

0x20000000 ┌──────────────────┐ 128KB
           │  SRAM1           │ ← 主RAM，DMA可访问
           │  (通用用途)      │
0x2001FFFF └──────────────────┘

0x10000000 ┌──────────────────┐ 64KB
           │  CCMRAM          │ ← 核心耦合内存
           │  (Core Coupled)  │   DMA不可访问！
0x1000FFFF └──────────────────┘

0x08000000 ┌──────────────────┐ 512KB
           │  FLASH           │ ← 程序存储
           │                  │
0x0807FFFF └──────────────────┘

0x40000000 ┌──────────────────┐
           │  外设寄存器区     │ ← AHB/APB外设
           │  (Peripheral)    │
0x5FFFFFFF └──────────────────┘
```

### 2.1.2 各区域特性

| 区域   | 起始地址   | 大小  | 特点                             |
| ------ | ---------- | ----- | -------------------------------- |
| SRAM1  | 0x20000000 | 128KB | 通用RAM，DMA可访问，默认数据区   |
| CCMRAM | 0x10000000 | 64KB  | 仅CPU可访问，DMA不可访问，速度快 |
| FLASH  | 0x08000000 | 512KB | 程序存储，支持读加速             |

### 2.1.3 链接脚本示例（F407）

```txt
/* 内存区域定义 */
MEMORY
{
    RAM (xrw)    : ORIGIN = 0x20000000, LENGTH = 128K
    CCMRAM (xrw) : ORIGIN = 0x10000000, LENGTH = 64K
    FLASH (rx)   : ORIGIN = 0x08000000, LENGTH = 512K
}

/* 段定义 */
SECTIONS
{
    /* 中断向量表 - 必须放在Flash起始位置 */
    .isr_vector :
    {
        . = ALIGN(4);
        KEEP(*(.isr_vector))
        . = ALIGN(4);
    } >FLASH

    /* 代码段 */
    .text :
    {
        . = ALIGN(4);
        *(.text)
        *(.text*)
        . = ALIGN(4);
        _etext = .;
    } >FLASH

    /* 只读数据 */
    .rodata :
    {
        . = ALIGN(4);
        *(.rodata)
        *(.rodata*)
        . = ALIGN(4);
    } >FLASH

    /* 已初始化数据 - LMA在Flash，VMA在RAM */
    .data :
    {
        . = ALIGN(4);
        _sdata = .;
        *(.data)
        *(.data*)
        . = ALIGN(4);
        _edata = .;
    } >RAM AT> FLASH

    /* 未初始化数据 */
    .bss :
    {
        . = ALIGN(4);
        _sbss = .;
        *(.bss)
        *(.bss*)
        _ebss = .;
    } >RAM

    /* CCMRAM段（可选使用） */
    .ccmram :
    {
        . = ALIGN(4);
        _sccmram = .;
        *(.ccmram)
        *(.ccmram*)
        . = ALIGN(4);
        _eccmram = .;
    } >CCMRAM AT> FLASH
}
```

## 2.2 STM32H7内存架构（复杂）

### 2.2.1 内存映射

```
地址空间布局（STM32H723）

0x20000000 ┌──────────────────┐ 128KB
           │  DTCM-RAM        │ ← 数据TCM，CPU专用
           │  (Data TCM)      │   DMA1不可访问！
0x2001FFFF └──────────────────┘

0x00000000 ┌──────────────────┐ 64KB
           │  ITCM-RAM        │ ← 指令TCM
           │  (Instruction)   │   可存放关键代码
0x0000FFFF └──────────────────┘

0x24000000 ┌──────────────────┐ 512KB
           │  AXI SRAM        │ ← AXI总线SRAM
           │                  │   DMA可访问
0x2407FFFF └──────────────────┘

0x30000000 ┌──────────────────┐ 128KB
           │  SRAM1           │ ← AHB SRAM
0x3001FFFF ├──────────────────┤   DMA1可访问！
           │  SRAM2           │ 128KB
0x3003FFFF ├──────────────────┤
           │  SRAM3           │ 32KB
0x30047FFF ├──────────────────┤
           │  SRAM4           │ 16KB
0x3004BFFF └──────────────────┘

0x38000000 ┌──────────────────┐ 64KB
           │  Backup SRAM     │ ← 备份域SRAM
0x3800FFFF └──────────────────┘

0x08000000 ┌──────────────────┐ 1MB
           │  FLASH           │
0x080FFFFF └──────────────────┘
```

### 2.2.2 各区域特性

| 区域     | 起始地址   | 大小  | 总线 | DMA访问              |
| -------- | ---------- | ----- | ---- | -------------------- |
| DTCM-RAM | 0x20000000 | 128KB | TCM  | DMA1不可，DMA2需BDMA |
| ITCM-RAM | 0x00000000 | 64KB  | TCM  | DMA不可              |
| AXI SRAM | 0x24000000 | 512KB | AXI  | MDMA可访问           |
| SRAM1    | 0x30000000 | 128KB | AHB  | DMA1/DMA2可访问      |
| SRAM2    | 0x30020000 | 128KB | AHB  | DMA1/DMA2可访问      |
| SRAM3    | 0x30040000 | 32KB  | AHB  | DMA1/DMA2可访问      |
| SRAM4    | 0x38000000 | 64KB  | AHB  | DMA2/BDMA可访问      |

### 2.2.3 关键问题：DMA访问限制

**问题描述：**

- STM32H7默认将变量放在DTCM-RAM（0x20000000）
- DTCM-RAM与DMA1控制器不在同一总线
- DMA1无法访问DTCM-RAM中的数据

**解决方案：**
将DMA缓冲区放到SRAM1/2/3（0x30000000区域）

### 2.2.4 链接脚本示例（H7）

```txt
/* 内存区域定义 */
MEMORY
{
    ITCMRAM (xrw)  : ORIGIN = 0x00000000, LENGTH = 64K
    DTCMRAM (xrw)  : ORIGIN = 0x20000000, LENGTH = 128K
    RAM_D2 (xrw)   : ORIGIN = 0x30000000, LENGTH = 288K  /* SRAM1+2+3 */
    RAM_D3 (xrw)   : ORIGIN = 0x38000000, LENGTH = 64K   /* SRAM4 */
    FLASH (rx)     : ORIGIN = 0x08000000, LENGTH = 1024K
}

/* DMA缓冲区段定义 */
SECTIONS
{
    /* 其他段... */

    /* DMA缓冲区 - 放到RAM_D2（DMA1可访问） */
    .dma_buffer (NOLOAD) :
    {
        . = ALIGN(4);
        _sdma_buffer = .;
        *(.dma_buffer)
        *(.dma_buffer*)
        . = ALIGN(4);
        _edma_buffer = .;
    } >RAM_D2
}
```

## 2.3 链接脚本语法详解

### 2.3.1 基本结构

```txt
/* ==================== 入口点定义 ==================== */
ENTRY(Reset_Handler)    /* 程序开始执行的位置 */

/* ==================== 栈和堆定义 ==================== */
_estack = ORIGIN(RAM) + LENGTH(RAM);  /* 栈顶 = RAM末尾 */
_Min_Heap_Size = 0x200;    /* 最小堆大小 */
_Min_Stack_Size = 0x400;   /* 最小栈大小 */

/* ==================== 内存区域定义 ==================== */
MEMORY
{
    /* 名称 (属性) : 起始地址 = ORIGIN, 大小 = LENGTH */

    /* 属性说明：
       R - 可读    W - 可写    X - 可执行
       A - 可分配  I - 可初始化
       rx  = 只读可执行（Flash）
       xrw = 可读可写可执行（RAM）
    */

    RAM (xrw)   : ORIGIN = 0x20000000, LENGTH = 128K
    FLASH (rx)  : ORIGIN = 0x08000000, LENGTH = 512K
}

/* ==================== 段定义 ==================== */
SECTIONS
{
    /* 段内容... */
}
```

### 2.3.2 段定义语法

```txt
SECTIONS
{
    /* ========== 基本段定义 ========== */
    .section_name :
    {
        /* 内容 */
    } >MEMORY_REGION

    /* ========== 位置计数器 ========== */
    . = ALIGN(4);           /* 4字节对齐 */
    . = 0x08000100;         /* 设置到特定地址 */

    _symbol = .;            /* 定义符号（可用于C代码） */

    /* ========== 输入段匹配 ========== */
    *(.text)                /* 所有.text段 */
    *(.text*)               /* 所有.text开头的段 */
    *(.text .text.*)        /* 多个模式 */

    /* ========== KEEP指令 ========== */
    KEEP(*(.isr_vector))    /* 防止被链接器优化掉 */

    /* ========== 双地址：VMA和LMA ========== */
    .data :
    {
        _sdata = .;
        *(.data)
        _edata = .;
    } >RAM AT> FLASH        /* VMA=RAM, LMA=Flash */

    /*
     * VMA (Virtual Memory Address): 运行时的地址
     * LMA (Load Memory Address):    加载时的地址
     *
     * .data段：
     *   - LMA = Flash（初始值存储位置）
     *   - VMA = RAM（运行时位置）
     *   - 启动代码负责从LMA复制到VMA
     */

    /* ========== LOADADDR函数 ========== */
    _sidata = LOADADDR(.data);  /* 获取段的LMA地址 */

    /* ========== NOLOAD属性 ========== */
    .bss (NOLOAD) :
    {
        _sbss = .;
        *(.bss)
        _ebss = .;
    } >RAM
    /* NOLOAD: 不从Flash加载，启动代码直接清零 */

    /* ========== 丢弃段 ========== */
    /DISCARD/ :
    {
        libc.a ( * )        /* 丢弃标准库中不需要的部分 */
        libm.a ( * )
    }
}
```

| ------------------- | -------------------- | ----------------- |
| `ORIGIN(memory)`    | 获取内存区域起始地址 | `ORIGIN(RAM)`     |
| `LENGTH(memory)`    | 获取内存区域大小     | `LENGTH(FLASH)`   |
| `LOADADDR(section)` | 获取段的LMA地址      | `LOADADDR(.data)` |
| `ADDR(section)`     | 获取段的VMA地址      | `ADDR(.data)`     |
| `SIZEOF(section)`   | 获取段大小           | `SIZEOF(.text)`   |
| `ALIGN(exp)`        | 对齐到exp边界        | `ALIGN(4)`        |

### 2.3.4 特殊段说明

```txt
/* ========== 中断向量表 ========== */
.isr_vector :
{
    . = ALIGN(4);
    KEEP(*(.isr_vector))    /* 必须保留，且放在Flash起始位置 */
    . = ALIGN(4);
} >FLASH

/* ========== ARM/Thumb转换 ========== */
.glue_7 :                   /* ARM调用Thumb函数的转换代码 */
{
    *(.glue_7)
    *(.glue_7t)
} >FLASH

/* ========== C++构造/析构函数表 ========== */
.preinit_array :
{
    PROVIDE_HIDDEN(__preinit_array_start = .);
    KEEP(*(.preinit_array*))
    PROVIDE_HIDDEN(__preinit_array_end = .);
} >FLASH

.init_array :
{
    PROVIDE_HIDDEN(__init_array_start = .);
    KEEP(*(SORT(.init_array.*)))
    KEEP(*(.init_array*))
    PROVIDE_HIDDEN(__init_array_end = .);
} >FLASH

/* ========== RAM中运行的函数 ========== */
.RamFunc :
{
    . = ALIGN(4);
    *(.RamFunc)
    *(.RamFunc*)
    . = ALIGN(4);
} >RAM AT> FLASH
```

## 2.4 自定义段的实现

### 2.4.1 定义段属性宏

```c
// 内存属性定义宏
#define DMA_BUFFER    __attribute__((section(".dma_buffer")))
#define CCM_RAM       __attribute__((section(".ccmram")))
#define FAST_CODE     __attribute__((section(".RamFunc")))
#define FLASH_CONST   __attribute__((section(".rodata")))

// 对齐属性
#define ALIGN_4       __attribute__((aligned(4)))
#define ALIGN_32      __attribute__((aligned(32)))
#define ALIGN_CACHE   __attribute__((aligned(32)))  /* 缓存行对齐 */
```

### 2.4.2 使用示例

```c
#include <stdint.h>

/* ========== DMA缓冲区（放到DMA可访问区域） ========== */
// 对于STM32H7，放到RAM_D2（0x30000000）
uint8_t uart_rx_buffer[256] DMA_BUFFER;
uint8_t spi_tx_buffer[128] DMA_BUFFER;

/* ========== CCMRAM变量（快速访问，DMA不可访问） ========== */
// 对于STM32F407，放到CCMRAM（0x10000000）
int fast_lookup_table[1024] CCM_RAM;
float pid_state[4] CCM_RAM;

/* ========== 在RAM中运行的函数（加速执行） ========== */
// 适合频繁调用的关键函数
void Critical_Control_Loop(void) FAST_CODE;

/* ========== Flash中的常量数据 ========== */
const uint8_t font_table[2048] FLASH_CONST;
const char* const error_messages[] FLASH_CONST = {
    "Error 1",
    "Error 2",
};
```

### 2.4.3 链接脚本配套定义

```txt
/* 在SECTIONS中添加自定义段 */

/* DMA缓冲区段 */
.dma_buffer (NOLOAD) :
{
    . = ALIGN(4);
    _sdma_buffer = .;
    *(.dma_buffer)
    *(.dma_buffer*)
    . = ALIGN(4);
    _edma_buffer = .;
} >RAM_D2     /* STM32H7: 放到DMA1可访问的区域 */

/* CCMRAM段 */
.ccmram :
{
    . = ALIGN(4);
    _sccmram = .;
    *(.ccmram)
    *(.ccmram*)
    . = ALIGN(4);
    _eccmram = .;
} >CCMRAM AT> FLASH

/* RAM函数段 */
.RamFunc :
{
    . = ALIGN(4);
    _sRamFunc = .;
    *(.RamFunc)
    *(.RamFunc*)
    . = ALIGN(4);
    _eRamFunc = .;
} >RAM AT> FLASH

_sRamFuncLoad = LOADADDR(.RamFunc);
```

### 2.4.4 启动代码支持

```c
// 如果使用自定义的初始化段，需要在启动代码中添加初始化

// 初始化CCMRAM（如果有初始值）
void Init_CCMRAM(void) {
    uint32_t *src = &_sccmram_load;
    uint32_t *dst = &_sccmram;
    while (dst < &_eccmram) {
        *dst++ = *src++;
    }
}

// 初始化RAM函数
void Init_RamFunc(void) {
    uint32_t *src = &_sRamFuncLoad;
    uint32_t *dst = &_sRamFunc;
    uint32_t size = (uint32_t)&_eRamFunc - (uint32_t)&_sRamFunc;
    memcpy(dst, src, size);
}
```

## 2.5 内存使用分析

### 2.5.1 编译后查看内存使用

```bash
# 使用arm-none-eabi-size工具
arm-none-eabi-size output.elf

# 输出示例：
#    text    data     bss     dec     hex filename
#   45678    1234    8900   55812    da04 output.elf

# 字段说明：
# text  = 代码 + 只读数据（Flash）
# data  = 已初始化数据（Flash存储 + RAM运行）
# bss   = 未初始化数据（只占RAM）
```

### 2.5.2 内存使用计算

```
Flash使用 = text + data
RAM使用  = data + bss + 栈 + 堆

示例：
Flash = 45678 + 1234 = 46912 字节 ≈ 45.8 KB
RAM   = 1234 + 8900 + 栈(1K) + 堆(512) ≈ 11.6 KB
```

## 附录：常见问题

### Q1: 为什么全局变量初始化为0和未初始化效果一样？

两者都放在.bss段，启动时都会被清零。但显式初始化为0更清晰。

### Q2: CCMRAM有什么用？

- 存放频繁访问的数据（如PID状态）
- 存放不需要DMA的缓冲区
- 相比主RAM，访问延迟更低

### Q3: 为什么字符串字面量在Flash？

字符串字面量是常量，放在.rodata段。这样可以节省RAM空间。

### Q4: 如何让函数在RAM中运行？

```c
__attribute__((section(".RamFunc")))
void MyFastFunction(void) {
    // 这个函数会被复制到RAM执行
    // 适合Flash访问慢或需要频繁执行的场景
}
```

### Q5: H7的DMA问题如何排查？

1. 检查缓冲区地址是否在DMA可访问区域
2. 使用 `DMA_BUFFER` 宏将缓冲区放到正确区域
3. 确认链接脚本中 `RAM_D2` 的定义正确
