# 项目架构设计方案

本文档描述 test_my_frame 项目的分层架构设计原则、数据流机制和封装规范，适用于跨层设计参考。

---

## 一、整体架构

### 1.1 分层结构

```
┌─────────────────────────────────────────────────────────────┐
│                     APP 层（应用层）                         │
│  - 业务逻辑                                                  │
│  - 任务调度                                                  │
│  - 消息处理                                                  │
│  - FreeRTOS 对象管理（队列、互斥量、信号量）                 │
└─────────────────────────────────────────────────────────────┘
                          ↓ 调用 DRV 接口
┌─────────────────────────────────────────────────────────────┐
│                     DRV 层（驱动层）                         │
│  - 外部模块驱动（电机、IMU、遥控器等）                       │
│  - 协议解析                                                  │
│  - 数据处理                                                  │
│  - 回调传递                                                  │
└─────────────────────────────────────────────────────────────┘
                          ↓ 调用 BSP 接口
┌─────────────────────────────────────────────────────────────┐
│                     BSP 层（板级支持层）                     │
│  - MCU 外设封装（CAN、UART、SPI、GPIO 等）                   │
│  - 中断回调分发                                              │
│  - 实例管理                                                  │
└─────────────────────────────────────────────────────────────┘
                          ↓ 调用 HAL 库
┌─────────────────────────────────────────────────────────────┐
│              HAL 层 + SEGGER RTT + FreeRTOS                  │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 层级调用原则

| 规则 | 说明 |
|------|------|
| **单向依赖** | 上层调用下层，下层不能调用上层 |
| **禁止跨层** | APP 不能直接调用 HAL，必须通过 DRV → BSP |
| **接口隔离** | 每层只暴露必要的接口，隐藏实现细节 |
| **配置向下** | 上层配置驱动下层，下层不依赖上层配置文件 |

### 1.3 各层职责

| 层级 | 职责 | 关键点 |
|------|------|--------|
| APP | 业务逻辑、任务调度 | 调用 DRV 接口，管理 RTOS 对象，使用DEF宏定义实例 |
| DRV | 模块驱动、协议解析 | 封装外部模块，提供解析函数，值拷贝BSP实例 |
| BSP | 外设封装、中断分发 | 定义DEF宏，管理实例数组，提供板级映射 |
| HAL | 硬件抽象层 | 包含多开发板配置，CubeMX 生成的基础代码 |

## 二、信息流设计

### 2.1 中断数据流

中断发生时的数据流向（必须先注册，才能回调）：

```
[MPU] interrupt
   │
   ↓ HAL 回调函数
   │
[BSP] hal_callback
   │    └─ 拷贝原始数据，查找实例，调用其回调
   ↓
[DRV] bsp_callback
   │    └─ 继续回调到 APP 层
   ↓
[APP] drv_callback
   │    └─ 拷贝原始消息，塞入队列（不做解析！）
   ↓
(interrupt finish) 中断结束
```

**重要原则：中断中不解析数据，只拷贝和传递**

### 2.2 任务处理流

消息处理线程中的数据流向：

```
[APP] 从队列获取原始消息
   │
   ↓ 解析数据（协议解析在此进行）
   │
[APP] 更新模块状态，执行业务逻辑
   │
   ↓ 调用 DRV 层接口
   │
[DRV] 调用 BSP 层接口
   │
   ↓
[BSP] 调用 HAL 库操作硬件
```

## 三、面向对象封装原则

### 3.1 核心原则

| 原则 | 实现方式 | 目的 |
|------|----------|------|
| **接口暴露最小化** | 只在 .h 中声明必要的公开函数 | 隐藏实现细节 |
| **禁止全局变量** | 使用 static 限定文件作用域 | 减少耦合 |
| **线程安全通信** | 使用队列、信号量、互斥量 | 避免竞态条件 |

### 3.2 文件组织规范

```
模块目录/
├── module.h          # 公开接口声明 + 结构体定义
├── module.c          # 实现 + 静态函数
└── module_def.h      # 类型定义（可选，供内部使用）
```

## 四、实例注册机制

采用"DEF宏定义实例 + 注册函数"模式，实现静态分配：

```c
// APP层：使用DEF宏定义实例
DEF_MOTOR(motor1, CAN_ID_0x201);

// 注册到BSP层
MotorRegister(&motor1);

// 直接访问成员
motor1.ref = 100.0f;
float speed = motor1.speed;
```

## 五、配置传递机制

```
┌─────────────────────────────────────────────────────────────┐
│                     APP 层                                   │
│  - app_cfg.h 定义开发板选择和应用参数                        │
│  - 通过初始化参数传递配置到下层                              │
└─────────────────────────────────────────────────────────────┘
                          ↓ 包含 app_cfg.h
┌─────────────────────────────────────────────────────────────┐
│                     BSP 层                                   │
│  - bsp_cfg.h 根据 DEVELOPMENT_BOARD 自动适配                 │
│  - 定义板载资源枚举、硬件映射、实例数量                       │
└─────────────────────────────────────────────────────────────┘
                          ↓ 无依赖
┌─────────────────────────────────────────────────────────────┐
│                     DRV 层                                   │
│  - drv_types.h 定义跨层通用类型                              │
│  - 不涉及硬件配置                                            │
└─────────────────────────────────────────────────────────────┘
```

## 六、RTOS 接口规范

**APP 层统一使用 FreeRTOS 原生 API，不使用 CMSIS-RTOS 封装层**

| 功能 | FreeRTOS 原生 API |
|------|-------------------|
| 创建任务（静态） | `xTaskCreateStatic()` |
| 删除任务 | `vTaskDelete()` |
| 延时 | `vTaskDelay(pdMS_TO_TICKS(ms))` |
| 创建队列（静态） | `xQueueCreateStatic()` |
| 发送队列 | `xQueueSend()` |
| 接收队列 | `xQueueReceive()` |
| 创建互斥量（静态） | `xSemaphoreCreateMutexStatic()` |
| 获取互斥量 | `xSemaphoreTake()` |
| 释放互斥量 | `xSemaphoreGive()` |

---

## 七、DEF宏定义规范

**DEF宏定义在底层（BSP层），APP层调用宏创建实例**

```c
// BSP层定义宏
#define DEF_GPIO(name, port, pin)              \
    static GPIOInstance name = {               \
        .GPIOx = port,                         \
        .GPIO_Pin = pin,                       \
        .state = GPIO_PIN_RESET,               \
    };

// APP层使用宏
DEF_GPIO(motor_in1, GPIOB, GPIO_PIN_12);
```

---

## 八、DRV访问BSP方式

**DRV结构体直接包含BSP结构体（值拷贝），不使用指针**

| 方式 | 定义 | 生命周期 | 内存安全 |
|------|------|----------|----------|
| 指针方式 ❌ | `GPIOInstance *gpio;` | 需手动管理 | 悬空风险 |
| 值拷贝方式 ✅ | `GPIOInstance gpio;` | 跟随DRV实例 | 安全 |

---

## 九、回调注册与分发

```
APP 调用 DRV 注册函数
       │
       ↓
DRV 调用 BSP 注册函数，传入回调
       │
       ↓
BSP 保存实例和回调指针
       │
       ↓
中断发生时，BSP 查找实例并调用回调
```

---

## 十、线程安全通信

| 队列类型 | 创建位置 | 说明 |
|----------|----------|------|
| 线程间通信队列 | `app/Src/robot.c` | 用于任务间传递消息 |
| 中断到自己的队列 | 任务自己文件中 | 用于中断向本任务传递数据 |

**队列长度统一设置为 1**

---

## 十一、通信方式选择

| 通信方式 | 核心功能 | 关键特点 | 适用场景 |
|----------|----------|----------|----------|
| **队列** | 任务间传递消息 | 数据拷贝，FIFO | 中断到任务，大量数据 |
| **互斥量** | 保护共享资源 | 优先级继承 | 共享数据访问 |
| **二值信号量** | 任务同步 | 轻量通知 | 事件通知 |
| **计数信号量** | 资源计数 | 多资源管理 | 资源池 |
| **任务通知** | 直接通信 | 最轻量 | 任务间简单同步 |
