# 代码要求

> 代码审查时要逐文件检查以下要点

---

## 一、正确性

1. 根据 HAL 库检查 BSP
2. 根据 BSP 和硬件手册检查 DRV

## 二、健壮性

1. BSP、DRV 层的 Register 函数参数检查
2. 全局变量要有初始值

## 三、可测试性、可移植性

1. DRV 层要留调参接口
2. 不修改 BSP 和 DRV 能写一个新的 APP
3. 通过写入结构体配置
4. 通过可覆盖的宏定义确定参数

## 四、运行安全、运行效率

（待补充详细要点）

## 五、简洁性

1. API 简洁

## 六、可读性

1. 变量命名规范
2. 注释、文档齐全
